# 需求与设计: 更新块 (Update Block) 结构增强与 Doctor 修复

## 1. 背景 (Context)
当前的 `UPDATE_BLOCK` 机制采用单点标记：
```html
<!-- UPDATE_BLOCK_START: 2026-04-19 22:28:46 | reason: 理由 -->
内容
<!-- UPDATE_BLOCK_END -->
```
虽然能够成功定界，但为了更清晰地匹配和避免潜在的块重叠（或结束标签丢失/混淆），我们希望在 `UPDATE_BLOCK_END` 也带上与 `START` 一致的时间戳。

同时，针对历史长河中可能出现的不标准或丢失了时间的“老旧块”，我们需要通过 `doctor` 提供一个自愈命令，使用文件的最后修改时间来推测并重构这些块。

## 2. 目标 (Objectives)
- **强化定界**: `UPDATE_BLOCK_END` 必须包含和 `START` 一致的时间戳。
- **平滑兼容**: 修改正则解析器，使其同时兼容含有和不含有时间戳的 `END` 标签，方便后续处理。
- **自动治愈**: 在 `doctor` 中增加 `--fix-update-blocks` 诊断项，扫描所有疑似更新块的内容，将缺失时间戳的旧块转化为标准的新块。时间未知时使用文件 `mtime`，每遇到一个未知块则 `+1s` 生成唯一时间戳。
- **功能验证**: 验证 `create-entity` 的 `--content` 能否正确触发内部正文和 WikiLink 的自动名称标准化（之前已在框架中拦截，现在需编写确切测试案例证明生效）。

## 3. 设计方案 (Proposed Design)

### 3.1 改造 `UpdateBlockManager` (`utility/schema_define.py`)
- **`BLOCK_PATTERN`**: 修改正则的结束部分以支持可选的时间戳匹配。
  新正则：`r"<!-- UPDATE_BLOCK_START: (.*?) \| reason: (.*?) -->\n(.*?)\n<!-- UPDATE_BLOCK_END(?:|: .*?) -->"`
- **`create_block(content, reason, timestamp=None)`**:
  - 新增一个可选的 `timestamp` 参数，如果未提供，则自动生成 `datetime.now()`。这为 `doctor` 重构旧块提供了覆盖时间的接口。
  - 生成格式变为：
    ```html
    <!-- UPDATE_BLOCK_START: {timestamp} | reason: {reason} -->
    {content}
    <!-- UPDATE_BLOCK_END: {timestamp} -->
    ```

### 3.2 扩展 `memocli doctor` (`commands/doctor.py`)
- **新增参数**: `--fix-update-blocks`
- **新增逻辑**: `_run_fix_update_blocks_pass(entity_files, issues, do_fix)`
  - 遍历所有 `entity_files` 的内容。
  - 使用更宽松的正则 `r"<!-- UPDATE_BLOCK_START.*?(?:(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})|).*?(?:reason:\s*(.*?)|) -->\n(.*?)\n<!-- UPDATE_BLOCK_END.*?-->"` 来捕获哪怕残缺的块。
  - 解析其中附带的 `timestamp`。如果缺失或不合法，则读取文件的 `mtime`（修改时间），将其格式化。若遇到多个缺失块，则让时间递增 `+1` 秒以保持唯一。
  - 提取 `content` 并利用 `UpdateBlockManager.create_block(content, reason, timestamp)` 将其替换为全新的标准块格式。
  - 如果匹配到了，但是本身已经是最新格式（包含匹配的 END timestamp），则跳过。
  - 如果被修改，则将新内容写回文件，并计入 `fixed_files`。

### 3.3 测试用例
在 `/tmp` 环境下执行：
```bash
memocli create-entity -p ./kb -n "Test-Links" -t "测试" -c "测试创建实体的同时带上 [[My Test Link]] 和 [[测试/页面|显示名字]]" -r 测试
cat ./kb/entities/Test-Links.md
```
期望看到：
1. `UPDATE_BLOCK_END` 带有时间。
2. `[[My Test Link]]` 被转成了 `[[My-Test-Link|My Test Link]]`。
3. `[[测试/页面|显示名字]]` 被转成了 `[[测试-页面|显示名字]]`。
