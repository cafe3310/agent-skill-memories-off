# 开发日志: 模型优先统一结果协议上线与全量工具迁移 (Agent-First Output & Migration)

## 1. 概述 (Overview)

为了追求极致的 Token 压缩和零解析成本，本任务彻底重构了 `memocli` 的输出协议。废弃了原有的混合输出模式，转而采用统一的 `<memocli-result>` XML 包装器作为唯一的输出接口，并实施了“信号与上下文分离”的错误报告策略。

### 1.1 核心设计决策 (Signal-Context Separation)
- **stderr (报错信号)**: 仅输出极简的错误摘要，用于满足 Shell 监控和 IDE 错误捕获。
- **stdout (XML 上下文)**: 所有的详尽错误消息、用法帮助和执行细节全部收拢在 XML 的 `<error-detail>` 节点中，供 Agent 纠错使用。
- **一致性**: 统一所有修改类操作必须包含 `-r/--reason` 参数以供审计。
