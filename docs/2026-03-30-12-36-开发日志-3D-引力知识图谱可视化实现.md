# 开发日志: 3D 引力知识图谱可视化实现 (3D-Gravity-KG)

## 1. 概述 (Overview)

用户提出需要一个类似 Obsidian 但具备「引力感」的 3D 知识库可视化界面。本任务旨在开发一个低干扰、自包含且具备语义重力的 3D 图谱工具。

### 1.1 来源文档与背景
- **设计文档**: `docs/2026-03-30-12-16-设计-3D-引力图谱可视化方案.md`
- **核心需求**: 
    - 实体为核心节点，别名为卫星节点且紧密绑定。
    - 相同类型具备引力（聚类），不同类型具备斥力（分层）。
    - 节点标签纯净化（不含类型前缀）。
    - 零服务端，单文件 HTML 自包含渲染。

---

## 2. 核心功能实现 (Key Features)

### 2.1 语义场模型 (Semantic Force Field)
- **类型向心力 (Type-Centric Gravity)**: 
    - 在每一帧物理仿真中，脚本动态计算同类型实体的实时质心（Centroid）。
    - 为每个实体施加一个指向其类型质心的向心力，促使同色节点在 3D 空间中形成自然的「领域云」。
- **别名锁定 (Alias Binding)**: 
    - 别名节点被赋予更小的质量（`val: 3` 对比实体的 `val: 10`）。
    - 别名与主实体间的连接强度（Link Strength）设为最高（1.0），且距离极短（10px），确保其紧紧贴合主实体。

### 2.2 数据解析与导出 (Python Logic)
- **脚本路径**: `memories-off-skill/scripts/graph.py`
- **智能 ID 标准化**: 自动剥离文件名中的类型前缀，统一使用 `{Type}-{Name}` 作为全局 ID。
- **关系全提取**: 
    - 解析 `relation as predicate: Target` 元数据。
    - 扫描正文 `[[WikiLinks]]` 建立隐式链接。
    - 自动重定向：即使指向别名，连线也会自动归集到其主实体上。

### 2.3 零依赖渲染 (Self-contained Rendering)
- **技术栈**: `3d-force-graph` (Vanilla JS 版本) + Three.js。
- **注入机制**: 
    - 解析结果被序列化为 JSON 字符串，直接硬编码到 HTML 模板的 JavaScript 常量中。
    - 通过 UMD 加载远程资源，避免本地 `node_modules` 负担。
- **零干扰分发**: 脚本在系统临时目录生成 `.html` 并调用 `webbrowser` 模块自动开启，完成后无需残留文件在仓库中。

### 2.4 交互与漫游
- **平滑对焦**: 实现 `onNodeClick` 逻辑，点击节点后相机平滑过渡并自动对齐。
- **视觉映射**:
    - `nodeAutoColorBy`: 节点颜色映射到实体类型。
    - `linkDirectionalArrowLength`: 显式关系带有方向箭头。
    - `linkCurvature`: 设置连线曲率，解决多重关联时的遮挡问题。

## 3. 技术挑战与修复记录 (Challenges & Fixes)

### 3.1 UMD 依赖冲突
- **问题**: 初版尝试使用 `react-force-graph-3d`，在单文件 HTML 环境下由于缺少 React 环境及 `forwardRef` 支持导致黑屏。
- **修复**: 切换至 `3d-force-graph` (Vanilla JS 版)，其内置了 Three.js 且不需要 React 核心库，完美适配 `file://` 协议下的单文件渲染。

### 3.2 ID 重复拼接
- **问题**: 历史遗留数据中的文件名包含类型前缀，导致 ID 出现 `宠物-宠物-咪咪` 的冗余。
- **修复**: 在解析器中增加 `_clean_display_name` 正则清洗逻辑，确保 ID 的唯一性与显示名称的纯洁度。
