# 需求: memories-off 外部文档仓库 (External Document Repository) 能力

## 1. 背景 (Background)
用户希望在 `memories-off` 知识库之外，能够快速检索和阅读现有的文档库（如 Obsidian 仓库、技术文档目录等），而无需将这些文档转化为 `memories-off` 的实体格式。

## 2. 定义与特性 (Definitions)
外部文档仓库 (External Document Repository) 遵循以下原则：
1. **只读性**: 该仓库是纯只读的，`memories-off` 不会修改其中的任何内容。
2. **文档而非实体**: 包含的是普通文档（Document），而非 `memories-off` 规范下的结构化实体（Entity）。
3. **被动检索**: 仅按用户需要和指示进行检索，`memories-off` 不会主动对该库进行探索或索引。

## 3. 目标 (Goals)
1.  在 `memories-off` 的库配置文件 `meta.md` 中支持配置外部文档仓库路径。
2.  新增 `find-doc-by-name` 工具，支持对外部文档仓库进行模糊检索。
3.  新增 `read-doc-by-name` 工具，支持读取外部文档仓库中的具体文档。
4.  支持通过配置过滤特定后缀名和排除特定子目录。

## 4. 需求细节 (Details)

### 4.1 配置项 (Configuration)
In `meta.md` 的 Frontmatter 中新增以下可选字段：
- `external document repo`: 外部文档仓库根目录。支持相对路径（相对于 `meta.md` 所在目录）或绝对路径。
- `external document extensions`: 要包含的文档扩展名列表（如 `.md`, `.txt`），多个扩展名以逗号分隔。
- `external document exclude dirs`: 要排除的子目录列表（如 `attachments`, `.git`），多个子目录以逗号分隔。

### 3.2 检索逻辑 (`find-doc-by-name`)
- **功能**: 根据用户提供的查询词，模糊匹配文档库中的文件名。
- **输入**:
    - `query`: 查询词。
- **输出**: 
    - 匹配到的文档列表。
    - 每个文档应包含：文档名（包含后缀，或仅文件名）、相对 `doc_root` 的路径、文档长度（字符数或行数）。
- **匹配规则**:
    - 仅匹配文件名部分。对于 `a/b/c/name.md`，使用 `name` 或 `name.md` 即可定位。
    - 应当遵循 `doc_extensions` 和 `doc_exclude_dirs` 的限制。

### 3.3 阅读逻辑 (`read-doc-by-name`)
- **功能**: 根据精确的文档名（包含路径或仅文件名）读取文档内容。
- **输入**:
    - `name`: 文档标识名。
- **输出**:
    - 文档的完整 Markdown 内容。
- **定位逻辑**:
    - 如果 `name` 能够唯一标识一个文件，则直接返回。
    - 如果存在同名文件，应返回错误或提示用户提供更精确的路径。

## 4. 技术约束 (Constraints)
- **只读性**: 这两个工具必须是纯只读的，不得修改外部文档库的内容。
- **性能**: 对于大型文档库（如数千个文件），检索应保持在可接受的时间内。
- **兼容性**: 必须兼容 macOS (darwin) 系统。

## 5. 测试策略 (Test Strategy)
- 准备一个包含嵌套目录、不同扩展名和需要排除目录的测试文档库。
- 验证 `meta.md` 配置的解析是否正确。
- 验证模糊检索的准确性和排除逻辑。
- 验证精确读取功能。
