# 开发日志: memocli 协议规范与帮助系统全面升级

## 1. 实现的功能点
- **memocli 协议规范 (v1)**：
    - 为 `scripts/` 下的所有 14 个业务脚本添加了 `--memo-cli-info` 参数，用于输出「一句话功能描述」和「调用示例」。
    - 添加了 `--memo-cli-call` 参数，用于标识脚本是否被 `memocli` 调用。
- **自适应帮助系统**：
    - 脚本现在能识别 `--memo-cli-call`，并自动将其 `argparse` 的 `epilog`（用法示例）从 `python3 script.py` 切换为 `memocli subcommand`。
- **增强型全局帮助**：
    - `memocli --help` 顶部现在显示所属的 Agent Skill 名称及 `SKILL.md` 的完整物理路径。
    - 全局帮助列表现在包含每个子命令的功能描述，对齐整齐。
    - 自动过滤了 `install.py`, `schema_define.py` 及 `__pycache__` 等内部项。
- **安装器升级**：
    - `install.py` 现在采用「动态探测」机制，在安装时遍历所有脚本并调用 `--memo-cli-info` 来生成静态的帮助内容，保证了运行时的响应速度。

## 2. 代码修改
- **批量修改**：更新了 `scripts/` 下的 14 个 Python 业务脚本。
- **重构**：`memories-off-skill/scripts/install.py`。
- **新增文档**：`docs/2026-03-29-22-20-设计-memocli-协议规范.md`。

## 3. 验证结果
- 全局帮助 `memocli --help` 输出格式完美，包含路径和描述。
- 子命令帮助 `memocli stats --help` 显示了包装器提示，且用法示例已自动更新为 `memocli` 格式。
- 业务功能保持不变。

## 4. 后续建议
- 目前描述内容写死在 Python 脚本中，未来可考虑从脚本的 docstring 中自动提取，以减少重复。
- 可以考虑为 `memocli` 添加一个 `update` 子命令，以便在添加新脚本后快速重新生成包装器。
