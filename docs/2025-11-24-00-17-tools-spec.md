# 工具规格说明 (Tool Specifications)

本文档详细定义了 `mcp-server-memories-off` 项目提供给上游（例如 LLM Agent）的所有工具。每个工具的定义包括其功能描述、开发状态以及精确的输入参数。

---

## 1. 实体管理工具 (Entity Management)

这类工具负责知识实体（Entity）的创建、读取、更新和删除（CRUD）等核心操作。

### addEntities
- **开发状态**: 已完成
- **功能描述**: 在指定的知识库中，根据提供的信息批量创建新的知识实体文档。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entity` (object | object[], required): 一个或多个实体对象。
        - `name` (string, required): 实体的唯一名称，将作为文件名。
        - `type` (string, optional): 实体的类型 (e.g., `person`, `concept`)。
        - `aliases` (string[], optional): 实体的别名列表。
        - `content` (string, optional): 实体的 Markdown 正文内容。
    - `reason` (string, optional): 本次操作的简要目的说明。

### readEntities
- **开发状态**: 待办 (对应 `TODO` 中的 `getEntity`)
- **功能描述**: 读取一个或多个知识实体的完整内容，包括 front matter 和正文。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityNames` (string[], required): 要读取的实体名称列表。
    - `reason` (string, optional): 本次操作的简要目的说明。

### deleteEntities
- **开发状态**: 已完成
- **功能描述**: 将一个或多个知识实体文件移入知识库的 `trash` 目录（软删除）。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityNames` (string | string[], required): 要删除的实体名称（单个、数组或逗号分隔的字符串）。
    - `reason` (string, optional): 本次操作的简要目的说明。

### renameEntity
- **开发状态**: 已完成
- **功能描述**: 重命名知识实体文件，并自动更新所有在其他实体中指向该实体的关系链接。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `oldName` (string, required): 实体的旧名称。
    - `newName` (string, required): 实体的新名称。
    - `reason` (string, optional): 本次操作的简要目的说明。

### mergeEntities
- **开发状态**: 已完成
- **功能描述**: 将多个源知识实体的内容和 Front Matter 合并到一个目标实体中，并删除源实体。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `sourceNames` (string[], required): 要合并的源实体名称列表。
    - `targetName` (string, required): 合并的目标实体名称。
    - `reason` (string, optional): 本次操作的简要目的说明。

### addEntityContent
- **开发状态**: 已完成
- **功能描述**: 在指定知识实体的指定章节末尾追加新的 Markdown 内容。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityName` (string, required): 要追加内容的实体名称。
    - `inSection` (string, required): 目标章节的标题。
    - `newContent` (string, required): 要追加的 Markdown 内容。
    - `reason` (string, optional): 本次操作的简要目的说明。

### deleteEntityContent
- **开发状态**: 已完成
- **功能描述**: 从指定知识实体的指定章节中精确删除一段 Markdown 内容。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityName` (string, required): 要删除内容的实体名称。
    - `inSection` (string, required): 目标章节的标题。
    - `contentToDelete` (string, required): 要精确删除的 Markdown 内容。
    - `reason` (string, optional): 本次操作的简要目的说明。

### replaceEntitySection
- **开发状态**: 已完成
- **功能描述**: 重写知识实体的指定章节，可同时修改其章节标题和全部正文内容。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityName` (string, required): 要替换章节的实体名称。
    - `oldHeading` (string, required): 旧章节的标题。
    - `newHeading` (string, required): 新章节的标题。
    - `newBodyContent` (string, required): 新章节的完整正文内容。
    - `reason` (string, optional): 本次操作的简要目的说明。

## 2. 关系管理工具 (Relation Management)

这类工具用于管理实体之间的关系。

### createRelations
- **开发状态**: 开发中 (In Progress)
- **功能描述**: 在一个或多个实体的 Front Matter 中批量添加关系。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `relations` (string | string[], required): 格式为 "source, verb, target" 的关系字符串或数组。
    - `reason` (string, optional): 本次操作的简要目的说明。

### deleteRelations
- **开发状态**: 已完成
- **功能描述**: 从一个或多个实体的 Front Matter 中批量删除关系。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `relations` (string | string[], required): 格式为 "source, verb, target" 的关系字符串或数组，用于精确匹配删除。
    - `reason` (string, optional): 本次操作的简要目的说明。

### garbageCollectRelations
- **开发状态**: 待办 (未实现)
- **功能描述**: 查找并清理知识库中所有指向不存在实体的“断裂链接”。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `dryRun` (boolean, optional, default: true): 是否为演习模式，只报告问题而不执行清理。
    - `reason` (string, optional): 本次操作的简要目的说明。

## 3. 检索与查询工具 (Retrieval & Query)

这类工具用于根据不同条件查询和检索知识。

### listEntities
- **开发状态**: 已完成
- **功能描述**: 使用 glob 模式匹配并列出知识库中的实体名称。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityNameGlobs` (string | string[], required): 用于匹配实体文件名的 glob 模式或模式列表。
    - `reason` (string, optional): 本次操作的简要目的说明。

### listEntityTypes
- **开发状态**: 已完成
- **功能描述**: 列出知识库中所有唯一的实体类型及其数量。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `reason` (string, optional): 本次操作的简要目的说明。

### getEntitiesToc
- **开发状态**: 已完成
- **功能描述**: 获取一个或多个知识实体文档的目录结构 (Table of Contents)。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityNames` (string | string[], required): 要获取目录的实体名称或列表。
    - `reason` (string, optional): 本次操作的简要目的说明。

### readEntitiesSections
- **开发状态**: 已完成
- **功能描述**: 精确读取一个或多个实体中特定章节的内容。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `entityNames` (string | string[], required): 实体名称或列表。
    - `sectionGlobs` (string | string[], required): 用于匹配章节标题的 glob 模式或列表。
    - `reason` (string, optional): 本次操作的简要目的说明。

### findEntitiesByMetadata
- **开发状态**: 部分完成 (存在 `TODO` 事项)
- **功能描述**: 根据元数据（Front Matter）键值对在实体中进行搜索。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `metadataQuery` (object, required): 用于搜索的键值对，值支持 glob 模式。
        - `entity type` (string, optional)
        - `date modified` (string, optional)
        - `date created` (string, optional) (注: 尚未实现)
        - `relation as` (string, optional) (注: 尚未实现)
        - `aliases` (string, optional)
    - `reason` (string, optional): 本次操作的简要目的说明。

### searchInContents
- **开发状态**: 已完成
- **功能描述**: 在所有实体正文（非 Front Matter）中进行 glob 模式匹配，并返回匹配的行。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `contentGlob` (string, required): 用于搜索内容的 glob 模式。
    - `reason` (string, optional): 本次操作的简要目的说明。

### find_relations
- **开发状态**: 待办 (未实现, 对应 `findEntitysByRelation`)
- **功能描述**: 在知识库中查找关系，可按目标实体或关系类型过滤。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `toEntity` (string, optional): 关系指向的目标实体名称。
    - `relationType` (string, optional): 关系类型。

### search_anywhere
- **开发状态**: 待办 (未实现)
- **功能描述**: 在文件名、元数据和正文中进行全局 glob 模式匹配。
- **输入参数**:
    - `libraryName` (string, required): 知识库的名称。
    - `pattern` (string, required): 用于全局搜索的 glob 模式。
