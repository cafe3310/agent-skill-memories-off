# 开发日志: memocli 协议规范与帮助系统全线贯通

## 1. 概述 (Overview)

基于用户 cafe3310 提出的「安装」功能需求，本阶段工作重点在于将 `memories-off-skill/scripts/` 下的分散脚本整合为一个统一的命令行入口 `memocli`。通过定义 `memocli` 协议 (v1) 并增强包装器逻辑，实现了高度自动化、自适应且具备 Agent 感知能力的 CLI 工具链。

## 2. 实现的功能点 (Key Deliverables)

### 2.1 memocli 协议规范 (Protocol v1)
- **元数据发现 (`--memo-cli-info`)**: 所有的 14 个业务 Python 脚本均已支持此参数，能够返回一句话描述和调用示例。这使得安装器能够动态生成包含所有子命令说明的全局帮助列表。
- **环境感知识别 (`--memo-cli-call`)**: 
    - 脚本现在能识别其是否由 `memocli` 调用。
    - **用法适配**: 当存在此参数时，脚本的 `argparse` 帮助文档及用法示例会自动从 `python3 script.py` 切换为 `memocli <subcommand>`。
    - **程序名统一 (`prog`)**: 通过动态设置 `argparse` 的 `prog` 参数，所有报错和用法提示均统一显示为 `memocli <subcommand>`，不再暴露底层实现细节。

### 2.2 增强型 memocli 包装器
- **Agent Skill 标识**: `memocli --help` 顶部现在显示所属 Skill 名称及 `SKILL.md` 的完整物理路径，方便追溯和规范对照。
- **智能路径注入**: 在包含 `meta.md` 的目录下执行时，若用户未显式提供 `--path` 或 `-p`，`memocli` 会自动追加 `--path .`，极大简化了常用操作。
- **自动化安装 (`install.py`)**: 替换了原有的 shell 脚本安装器，采用 Python 驱动。它在安装时会遍历所有脚本，通过协议获取描述并生成一个高度静态化的、响应极快的包装器。

### 2.3 文档与提示词同步
- **SKILL.md**: 增加了完整的「安装与命令行工具」章节，记录了协议和智能特性。
- **README.md**: 重构了快速开始和工具使用部分，确立了 `memocli` 作为首选交互方式的地位。
- **prompts/个人助手.md**: 更新了 Agent 提示词，明确指示其优先使用 `memocli` 并利用其路径自识别特性，以减少 Token 消耗。
