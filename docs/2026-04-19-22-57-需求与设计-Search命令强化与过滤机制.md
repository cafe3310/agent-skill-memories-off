# 需求与设计: Search 命令强化 (过滤与别名支持)

## 1. 背景 (Context)
在 `2026-04-12-00-52-设计-其他Agent的优化建议.md` 中，指出了 `search` 和列表过滤的不足：
- `list --type` 等功能缺失，导致为了按类型查找实体不得不写 Python 脚本。
- `--name` 搜索实体名时，只搜索文件名，不支持通过 `aliases` 搜索。
- `--meta` 搜索过于宽泛，无法精确定位特定的关系（Relation）。

综合这几点，我们决定重构 `search.py`，通过纯 Python 实现代替原本的 `grep` 子进程调用，从而获得精准的元数据解析能力。

## 2. 目标 (Objectives)
- **新增过滤参数**: 支持 `--type` (按实体类型，支持正则) 和 `--rel` (按关系目标，支持正则) 对搜索范围进行精准裁剪。
- **别名感知**: 当使用 `--name` (或默认的 `--all`) 搜索实体名时，必须同时匹配文件内部的 `aliases` 字段。
- **纯 Python 实现**: 移除 `subprocess.run(grep)`，改为利用 `MetadataParser` 解析文件后直接应用正则匹配。废弃意义不大的 `--meta` 参数。

## 3. 设计方案 (Proposed Design)

### 3.1 参数变更 (`search.py`)
- **新增**: `-t, --type`: 过滤实体类型，支持正则表达式 (例如 `-t "人物|组织"`)。
- **新增**: `--rel`: 过滤关系目标。格式必须为 `谓词模式:目标模式` (恰好一个冒号)，前后两部分均支持正则表达式。需要注意的是，由于关系是双向分布的（有些记在 A 文件里指向 B，有些记在 B 文件里指向 A），`--rel` 仅能过滤出**当前实体文件中明确包含该匹配声明**的实体，不具备图谱逆向推导能力。
- **移除**: `-m, --meta`: 被具体的 `--type` 和 `--rel` 替代。
- **保留**: `-n, --name` (仅搜索文件名和别名), `-c, --content` (仅正文), `-a, --all` (综合), `--names-only`, `-C, --context`。

### 3.2 核心执行逻辑
1. **预编译**: 将用户传入的各种 `pattern` 预编译为正则对象 `re.compile(..., re.IGNORECASE)`。
2. **全库遍历**: 遍历 `entities/*.md` 下的所有文件，读取内容并使用 `MetadataParser.split_content` 分离元数据与正文。
3. **第一道防线 (前置过滤)**:
    - 如果指定了 `--type`，判断 `metadata.get("entity type")` 是否匹配传入的正则，不符则直接跳过该文件。
    - 如果指定了 `--rel` (`pred_pattern:target_pattern`)：
        - 遍历 `metadata` 中所有以 `relation as ` 开头的键。
        - 如果 `pred_pattern` 匹配上该键的谓词部分（去掉 `relation as ` 后的部分），则进一步提取对应的目标列表（逗号分隔）。
        - 检查目标列表中是否有任何一项能够匹配 `target_pattern`。
        - 如果整个文件的所有关系声明都不能满足上述条件，则跳过该文件。
4. **第二道防线 (基础正则匹配)**:
    - 如果主 `pattern` 为空（用户可能只用 `--type` 或 `--rel` 来筛选列表），则该文件直接作为匹配项。
    - 如果主 `pattern` 不为空：
        - `name_match`: 判断文件名 (`file.stem`) 或 `aliases` 列表中是否包含能命中正则的文本。
        - `content_match`: 遍历 `body` 的每一行，判断是否命中正则。为了支持 `-C` (Context)，提取命中行附近的上下文。
        - 根据 `scope` (`name`, `content`, `all`) 决定是否采纳这个文件。
5. **输出格式**:
    - 如果是 `--names-only`，输出去重后的文件名列表。
    - 如果带详细信息，按照实体对匹配结果进行分组展示。

### 3.3 针对 `-C` (上下文) 的简易实现
- 提取正文 `lines`。
- 如果某行匹配，记录 `max(0, i-C)` 到 `min(len, i+C+1)` 之间的行号。
- 将收集到的行号去重并排序，然后按顺序输出，如果不连续则用 `---` 隔开。
