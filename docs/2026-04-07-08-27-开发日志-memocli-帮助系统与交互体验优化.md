# 开发日志: memocli 帮助系统与交互体验优化 (Memocli-UX-Enhance)

## 1. 概述 (Overview)

为了提升 Agent 和用户在 CLI 环境下的交互效率，本任务对 `memocli` 包装器的帮助系统进行了优化，使其更加符合标准的命令行工具行为，并显著降低了获取帮助信息的认知负荷。

### 1.1 来源文档与背景
- **设计文档**: `docs/2026-03-29-18-05-设计-CLI-性能与输出优化.md`
- **核心问题**: 
    - 之前的 `memocli` 在无参数或使用 `help` 子命令时无法正确触发全局帮助。
    - 帮助信息中的指令引导不够标准，容易产生混淆。

---

## 2. 核心改进 (Key Improvements)

### 2.1 增强型帮助触发逻辑
- **修改文件**: `memories-off-skill/scripts/install.py`
- **逻辑优化**: 
    - 允许 `memocli` (不带任何参数) 直接输出全局帮助。
    - 支持 `memocli help` 作为内置劫持命令，跳转至全局帮助输出。
    - 统一了 `--help` 和 `-h` 的处理逻辑。

### 2.2 帮助信息标准化
- **视觉引导**: 在全局帮助输出中，明确标注了常用指令。
- **一致性**: 将原本混杂的指令说明统一为 `--help` 形式，例如：
    - `memocli --help`: 显示全局帮助。
    - `memocli <subcommand> --help`: 显示特定子命令的详细用法。
