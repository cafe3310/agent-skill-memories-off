# 开发日志: 关系管理重构与多值关联支持

本次开发主要解决了显式关系管理中的单值覆盖 Bug，并引入了基于方向的角色化命名规范（rel-out / rel-in），提升了 Agent 在创建和更新实体时的关联效率。

## 1. 背景与问题
- **覆盖 Bug**: 原有的 `manage-relations` 在添加关系时直接覆盖 metadata 键值对，导致「一对多」关系丢失。
- **效率低下**: 建立双向关联需要多次调用子命令。
- **概念模糊**: `from` 和 `to` 参数在实际使用中极易混淆方向。

## 2. 核心变更

### 2.1 底层解析器增强 (`schema_define.py`)
- 在 `MetadataParser` 类中新增 `add_relation` 和 `remove_relation` 方法。
- **多值支持**: 采用逗号分隔的字符串存储同一谓词下的多个目标（如 `relation as member: Alice, Bob`）。
- **去重逻辑**: 在添加时自动标准化名称并去重，确保幂等性。

### 2.2 命名规范统一方案 A (`--rel-out` / `--rel-in`)
为了简化心智模型，所有涉及实体的命令均支持：
- `--rel-out`: **出站**。修改「当前中心实体」，指向目标。
- `--rel-in`: **入站**。修改「来源实体」，使其指向当前中心实体。

### 2.3 脚本功能全量升级
1. **`manage_relations.py`**:
   - 彻底重构，支持 `--rel-out` (Add/Remove) 和 `--rel-in`。
   - 实现跨文件修改逻辑，可一键将当前实体“挂载”到多个来源。
2. **`create_entity.py`**:
   - 增加 `--rel-out` 和 `--rel-in` 参数。
   - 允许在创建实体的瞬间完成双向图谱构建。
3. **`append_update.py`**:
   - 增加 `--rel-out` 和 `--rel-in` 参数。
   - 允许在 Buffer-Update（追加正文）的同时更新语义关联。

## 3. 验证结果
- **Bug 修复**: 经测试，连续添加不同目标已能正确在元数据中追加而非覆盖。
- **一致性**: 跨文件修改逻辑已能正确处理目标实体不存在的情况并给出 WARN。
- **语义直观**: Agent 调用 `--rel-out` 和 `--rel-in` 的逻辑方向与 `get-relations` 的查询方向完全对应。
