# 需求分析：实体时序管理与智能重组

## 背景与目的
用户希望对现有的知识库实体管理能力进行扩展，增加对时间维度的支持，并探索更智能的实体内容整理方式。这些需求主要围绕“时序化记录”、“时序化读取”和“智能化重组”展开。

## 核心需求项

### 1. 实体内容的 Block 风格与时序化
- **目标**：希望文档实体保持 block（块状）风格，并在每次追加/修改 block 时，自动记录并附加该 block 的发生/记录时间。
- **价值**：能够追踪实体内部每一块内容的历史沿革，使知识库具备更强的时间溯源能力和动态生长感。

### 2. 时序感知的实体内容读取
- **目标**：新增一个 subcommand，专门用于读取一个或多个实体的详细信息（包括章节列表、正文内容、元数据）。
- **关键特性**：支持传入时间维度的参数（可选时间信息），例如仅读取某个时间点之后新增的章节或带有特定时间戳的 block。

### 3. 基于时间切片的图谱全局读取
- **目标**：新增一个 subcommand，支持读取整个知识图谱在特定时间段内的状态切片。
- **潜在方案**：由于 `memories-off` 具备 Git 驱动的强制审计日志（Audit），可以考虑直接利用 Git 命令和 Commit 历史来实现。通过比对不同时间点的 commit，提取出该时间窗口内的图谱增量或全貌。

### 4. 智能化实体内容重组与整理
- **目标**：引入更智能的机制，对长期积累下变得冗长或产生矛盾的实体内容进行整理。
- **挑战与设想**：具体方案待定。一个极具潜力的研究方向是利用大语言模型（LLM）的推理能力，自动识别相互冲突的记录，进行逻辑调和（如将冲突部分标注或重组），或者将长串的时序 block 压缩提炼为更紧凑的结构化摘要，同时保留原始细节的链接。
