# 设计文档：memocli 全局帮助系统增强方案 (Enriched-Global-Help)

## 1. 背景 (Problem Statement)

目前 Agent (如 Gemini, Claude) 在初次接触 `memories-off` 知识库时，通过 `memocli --help` 只能获取到子命令列表和简短描述。为了了解具体参数格式（如“逗号分隔”还是“分号分隔”、“是否可重复”），Agent 必须逐个运行 `memocli <subcommand> --help`。

这种“往返式”交互增加了 Token 消耗并降低了任务执行效率。

---

## 2. 核心目标 (Core Objectives)

1.  **一站式获取信息**：在全局帮助中直接暴露每个子命令的 One-shot 示例。
2.  **降低易错率**：示例应包含 optional args、repeatable args 以及参数分隔符等关键易错点。
3.  **保持简洁**：通过合理的缩进和格式化，确保信息量增加的同时，CLI 输出依然易于阅读。

---

## 3. 实施方案 (Implementation Plan)

### 3.1 协议增强 (Script Protocol)
`ScriptBase` 已支持在 `--memo-cli-info` 下输出 `Example:`。
- **动作**：审核所有子命令脚本（`scripts/*.py`），确保其 `example` 字段包含高质量、覆盖全面的 One-shot 指令。
- **参数全称规范**：为了让 Agent 更准确地理解参数含义，示例中必须使用参数的全称（如 `--name`, `--type`, `--entity`），严禁在全局帮助示例中使用 `-n`, `-t`, `-e` 等简写。

### 3.2 安装器增强 (`install.py`)
更新 `create_memocli()` 函数：
1.  **提取 Example**：在循环扫描脚本时，同时提取 `Example:` 后缀的内容。
2.  **Bash 脚本构造**：
    -   使用更紧凑的输出模板。
    -   增加对 `Example` 的转义处理。
3.  **视觉优化**：
    ```text
    可用子命令:
      create-entity        - 创建一个新实体
        Example: memocli create-entity -n "Alice" -t "Person" --add-rel-out "member:TeamA,TeamB" -r "init"

      manage-relations     - 管理显式关系
        Example: memocli manage-relations -s "Alice" --add-rel-out "friend:Bob" --add-rel-out "work:Charlie"
    ```

### 3.3 重点脚本示例核查表
| 脚本 | 示例应体现的特性 |
| :--- | :--- |
| `create-entity` | `-n`, `-t`, `add-rel-out` (冒号+逗号分隔), `-r` |
| `manage-relations` | `--source`, `--add-rel-out` (可重复使用) |
| `append-update` | `--entity`, `--section`, `content` 直接输入或多行 |
| `search` | `pattern`, `--name` (仅搜名称), `--names-only` (简洁输出) |

---

## 4. 技术实现细节 (Bash Snippet)

在 `install.py` 生成的 `memocli` 脚本中，帮助部分将改为：

```bash
# ...
    echo "可用子命令:"
    echo "  <subcommand>         - <description>"
    echo "    Example: <one-shot-example>"
# ...
```

---

## 5. 预期收益 (Benefits)

1.  **Agent 友好型设计**：模型只需一次调用即可掌握整个工具链的使用精髓。
2.  **自文档化**：降低了人类开发者的心智负担，安装时即生成最新的、带示例的手册。
3.  **容错性**：明确了 `comma-separated` 等格式细节，减少参数解析错误的发生。
