# 设计文档: 子命令错误提示增强与多语言优化 (Subcommand-Error-UX)

## 1. 背景与问题 (Context & Problem Statement)

在当前的 Agent 协作流中，当 `memocli` 的子命令因参数错误（缺失、类型错误等）导致执行失败时，其反馈机制存在以下两个痛点，导致 Agent 必须花费额外的回合（Loop）来纠错：

1.  **语言不统一 (Language Barrier)**: 默认的 `argparse` 报错信息通常为英文（如 `error: the following arguments are required: -p/--path`），而项目整体采用中文协作。
2.  **帮助信息缺失 (Lack of Guidance)**: 报错后仅显示具体的错误原因，不显示该子命令的完整用法（Help/Usage）。Agent 需要再次运行 `--help` 才能理解正确的参数组合。

根据「极致压缩」和「手术式精准」的原则，我们需要一种机制，在报错的同时，将“错误原因 + 修正建议 + 完整用法”一次性喂给 Agent。

## 3. 优化方案 (Proposed Solution)

### 3.1 核心策略：劫持 argparse 错误处理
通过在 `schema_define.py` 的 `ScriptBase` 中自定义一个 `ArgumentParser` 子类，重写其 `error` 方法，实现以下逻辑：

1.  **翻译核心错误 (Translation)**: 映射常见的 `argparse` 错误信息至中文。
2.  **强制输出帮助 (Auto-Help)**: 在输出错误信息后，自动调用 `print_help()`，确保 Agent 立即获得修正参考。
3.  **XML 包裹 (XML Wrap)**: 将错误和帮助信息包裹在 `<error_report>` 标签中，明确告知 Agent「你应该停下来并根据以下信息修正」。

### 3.2 伪代码实现 (Pseudocode)

```python
class ChineseArgumentParser(argparse.ArgumentParser):
    def error(self, message):
        # 1. 常见翻译映射
        if "the following arguments are required" in message:
            args = message.split(":")[-1]
            message = f"缺少必要参数:{args}"
        elif "argument" in message and "invalid" in message:
            message = message.replace("argument", "参数").replace("invalid", "无效的")
        
        # 2. 构造输出
        print("\n[错误] 参数验证失败!")
        print(f"原因: {message}")
        print("-" * 20)
        self.print_help()
        
        # 3. 强制 XML 包裹
        print(f"\n<error_report fatal=\"true\" reason=\"{message}\">")
        print("  <instruction>请检查以上用法 (usage) 并重新构造指令。</instruction>")
        print("</error_report>")
        
        sys.exit(2)
```

### 3.3 集成到 ScriptBase
在 `ScriptBase._init_parser` 中，不再直接实例化 `argparse.ArgumentParser`，而是使用 `ChineseArgumentParser`。

## 4. 预期收益 (Expected Benefits)

- **纠错闭环**: Agent 在单词失败后，无需发起新的 `memocli <cmd> --help` 查询，直接在报错输出中即可获得正确语法。
- **Token 压缩**: 虽然单次报错输出变长，但省去了一次完整的「请求帮助-返回结果」的 Turn，总体 Token 消耗和延迟将降低。
- **一致性**: 统一了全库所有脚本的报错风格。

## 5. 验证标准 (Validation Criteria)

1.  运行一个故意缺少参数的指令（如 `memocli stats`，不加 `-p`），观察输出：
    - 是否包含中文报错？
    - 是否自动打印了帮助信息？
    - 是否包含 XML 包裹？
2.  Agent 是否能够基于该输出，在下一轮直接给出正确的修复指令。
