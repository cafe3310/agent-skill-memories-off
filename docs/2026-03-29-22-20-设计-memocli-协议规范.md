# 需求与设计: memocli 协议规范与帮助系统升级

## 1. 背景
用户 cafe3310 希望进一步提升 `memocli` 的可用性和信息密度。目前 `memocli --help` 仅显示子命令列表，缺乏必要的描述和上下文信息（如 Agent Skill 属性）。

## 2. 目标
- **增强帮助信息**：在 `memocli` 顶部显示其所属的 Agent Skill 及 `SKILL.md` 的完整路径。
- **提供命令说明**：为每个子命令（Python 脚本）提供一句话的功能描述。
- **自适应帮助输出**：通过新增的协议参数，使子命令能识别自身是被 `memocli` 调用，并相应调整帮助文档中的示例（Example）。

## 3. 设计规范 (memocli Protocol)

我们将为 `memories-off-skill/scripts/` 下的所有业务脚本引入两个核心参数：

### 3.1 `--memo-cli-info`
- **职能**：元数据查询。
- **预期输出**（打印至 `stdout`）：
  - 第一行：`Description: <一句话功能描述>`
  - 第二行：`Example: <基本的调用示例>`
- **应用场景**：`memocli` 在生成全局帮助列表时调用，以展示各子命令的用途。

### 3.2 `--memo-cli-call`
- **职能**：执行环境标识位。
- **影响**：
  - 当此参数存在时，脚本在处理 `--help` 时，应将帮助文档底部的「用法示例」中的 `python3 <script>.py` 替换为 `memocli <action>`。
  - 其他执行逻辑（如路径注入提示）可根据此标识位进行静默处理或样式调整。

## 4. `memocli` 包装器增强
- **头部信息**：
  ```
  Memories-Off CLI (memocli) - An Agent Skill Wrapper
  Skill Root: /Users/sipan/workspace/mcp-server-memories-off/memories-off-skill/SKILL.md
  --------------------------------------------------
  ```
- **命令列表逻辑**：
  - 遍历 `scripts/` 下的 `.py` 文件（排除 `install.py`, `schema_define.py`, `__init__.py` 等）。
  - 对每个文件调用 `python3 <script>.py --memo-cli-info` 获取描述。
  - 格式化输出为：`  <action>  - <description>`。

## 5. 实施计划
1.  **分批次更新脚本**：依次为 `stats.py`, `init.py`, `create_entity.py` 等添加参数支持。
2.  **更新 `install.py`**：重新生成支持新协议的 `memocli` 包装器。
3.  **验证**：检查 `memocli` 和 `memocli stats --help` 的输出是否符合预期。
