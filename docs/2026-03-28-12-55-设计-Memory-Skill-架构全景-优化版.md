# 2026-03-28-12-55-设计-Memory-Skill-架构全景-优化版

本文档定义了 `memory-skill` 的优化分层模型，旨在通过清晰的职责划分，实现一个轻量、严谨且具备全局审计能力的本地知识库方案。

---

## 1. 核心分层 (Core Layers)

### L1: 仓库定义 (Repository Layer)
*职责: 定义知识库作为一个整体的物理边界与全局规则。*
- **物理结构**:
    - `meta.md`: 全局手册，定义用途、Schema 和规则。
    - `journey.md`: **全局审计日志**，顺序记录库内所有变更。
    - `entities/`: 存放所有实体文件的目录。
    - `trash/`: 存放软删除文件的目录。
    - `backups/`: 存放压缩备份的目录。

### L2: 文件定义 (Thing Definition)
*职责: 定义知识库中三种核心文件（Things）的内部结构。*
- **Thing 分类**:
    - **Entity (实体)**: 承载具体领域知识。
    - **Journal (日志)**: 即 `journey.md`，追加式变更流。
    - **Meta (手册)**: 即 `meta.md`，全局配置与规则。
- **内部逻辑**:
    - **Front Matter**: YAML 格式元数据。
    - **Body**: Markdown 格式正文，由 `Section` (章节) 和 `Heading` (标题) 构成。

### L3: 关联定义 (Relation Layer)
*职责: 定义实体间的拓扑连接，构建知识图谱。*
- **显式关系**: `relation as [predicate]` (Front Matter)。
- **隐式链接**: WikiLinks `[[ThingName]]` (Body)。
- **维护逻辑**: 实体重命名时的全局链接同步。

### L4: 动作定义 (Action Layer)
*职责: 定义 Agent 可用的原子化工具集。*
- **动作分类**:
    1.  **观察 (Observation)**: 全局状态感知 (stats, load manual)。
    2.  **检索 (Discovery)**: 发现目标实体名 (search/find by name/metadata/content)。
    3.  **阅读 (Reading)**: 解析内容细节 (load full/toc/sections)。
    4.  **修改 (Mutation)**: 变更系统状态 (create, edit, rename, merge, trash, relation)。

---

## 2. 核心协议 (Core Protocols)

- **追加式全局审计 (Append-only Journaling)**: 所有的 `Mutation` 动作必须顺序追加到唯一的 `journey.md` 中。
- **唯一性匹配协议 (Uniqueness)**: 编辑操作必须基于在搜索范围内有且仅有一个匹配项。
- **标准化清洗协议 (Normalization)**: 对名称、标题和键名进行规范化，确保机器处理的一致性。
