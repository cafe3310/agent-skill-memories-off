# 开发日志: 模型优先的统一结果输出协议上线 (Agent-First Protocol)

## 1. 概述 (Overview)

为了追求极致的 Token 压缩和零解析成本，本任务彻底重构了 `memocli` 的输出协议。废弃了原有的混合输出模式，转而采用统一的 `<memocli-result>` XML 包装器作为唯一的输出接口。

### 1.1 背景与目标
- **目标**: 实现一套结构化、可审计、自包含的输出格式。
- **设计文档**: `docs/2026-04-07-22-58-设计-模型优先的统一结果输出协议.md`

---

## 2. 核心改进 (Key Improvements)

### 2.1 统一包装器 memocli-result
- **结构**: 引入 `<memocli-result>` 根标签，包含 `subcommand`, `reason`, `result` 三大核心属性。
- **追溯**: 新增 `<source-sh>` 节点，实时记录并还原完整的命令行调用。
- **内容**: 原本分散的执行逻辑输出现在统一归集到 `<content>` 节点中。
- **错误**: 失败时提供 `<error-detail>`，集成错误原因与修复建议。

### 2.2 审计理由 (Reason) 成为一等公民
- **增强**: 在 `ScriptBase` 层为所有子命令自动注入 `-r/--reason` 参数。
- **逻辑**: 该参数被强制要求在 XML 属性中体现，强化了 Agent 操作的可回溯性。

### 2.3 异常拦截升级
- **覆盖**: `CustomArgumentParser.error` 现已完美适配新协议。当参数校验失败时，Agent 将收到结构化的错误报告而非单纯的 stderr 文本。

---

## 3. 验证记录 (Validation)

### 3.1 成功路径
- [x] `memocli stats -p ... -r "理由"` -> 返回结构化 success 报告。

### 3.2 失败路径
- [x] 路径不存在 -> 返回 failed 报告 + `<error-detail>`。
- [x] 参数缺失/无效 -> `argparse` 报错内容被成功拦截并封装进 XML 报告。
