# 计划: 全量工具集适配统一结果输出协议 (Full-Suite-Migration)

## 1. 概述 (Overview)

在 `stats.py` 成功试点 `memocli-result` 协议后，我们需要将此「模型优先」的输出规范推向全量工具集。这不仅是为了降低 Token 消耗，更是为了确保 Agent 在执行任何操作（如创建、编辑、检索）时，都能获得一致的结构化反馈。

### 1.1 核心目标
- **完全解耦**: 移除脚本中直接使用 `print` 输出 Plain Text 的逻辑。
- **协议闭环**: 统一通过 `ScriptBase` 的 `add_result()` 暂存信息，并由 `finalize()` 统一封包。
- **审计强制**: 确保所有具有副作用的操作（创建、修改、删除）都必须提供 `-r/--reason` 参数。

---

## 2. 迁移列表与分类 (Migration List)

基于对 `memories-off-skill/scripts/` 的扫描，迁移任务分为以下三类：

### A 类：检索与观察 (低风险，快速迁移)
- [ ] `search.py`: 将 `grep` 结果重构为结构化的 `<content>` 列表。
- [ ] `inspect-schema.py`: 将元数据定义重构成紧凑的内容。
- [ ] `get-relations.py`: 提取实体关系图谱。

### B 类：副作用操作 (核心，需要理由审计)
- [ ] `create-entity.py`: 创建实体。
- [ ] `append-update.py`: 缓冲更新。
- [ ] `consolidate-updates.py`: 合并缓冲区。
- [ ] `rename.py` / `rename-type.py`: 重命名。
- [ ] `merge-entities.py` / `merge-types.py`: 合并。
- [ ] `manage-relations.py`: 关系管理。
- [ ] `init.py`: 库初始化。

### C 类：系统维护与集成 (特殊)
- [ ] `commit.py`: 封装 Git 提交。
- [ ] `audit-fix.py`: 审计与自动修复。
- [ ] `graph.py`: 3D 图谱生成（仅输出结果 XML，HTML 在浏览器开启）。

---

## 3. 实施计划 (Implementation Plan)

### 第一阶段：检索类工具 (Batch 1)
- **重点**: 确保存量 XML 逻辑（如 `search_results`）被平滑迁移到 `memocli-result/content` 内部。
- **目标文件**: `search.py`, `inspect_schema.py`, `get_relations.py`。

### 第二阶段：修改类工具 (Batch 2)
- **重点**: 统一参数解析器，并在报错时自动提供修复建议。
- **目标文件**: `create_entity.py`, `append_update.py`, `manage_relations.py`, `rename.py`。

### 第三阶段：高风险/复杂逻辑 (Batch 3)
- **重点**: 处理涉及多个文件变动和 Git 交互的逻辑。
- **目标文件**: `merge_entities.py`, `audit_fix.py`, `commit.py`。

## 4. 迁移模板 (Migration Pattern)

所有迁移应遵循以下代码范式：

```python
class MySubcommand(ScriptBase):
    def __init__(self):
        super().__init__(
            action_name="my-command",
            description="...",
            example="..."
        )
        # 添加自定义参数
        self.parser.add_argument("-x", ...)

    def run(self):
        self.setup() # 解析参数并初始化 ctx
        
        # 业务逻辑
        if success:
            self.add_result("操作成功描述")
            self.finalize(success=True)
        else:
            self.error("具体错误描述", instruction="修复建议")
```

## 5. 验证与交付
- 每完成一个 Batch，将进行批量测试：
    1. `memocli <cmd> --help`: 确认 usage 正确。
    2. `memocli <cmd> (缺少参数)`: 确认报错符合新协议。
    3. `memocli <cmd> (正常运行)`: 确认输出 XML 完整且语义正确。
