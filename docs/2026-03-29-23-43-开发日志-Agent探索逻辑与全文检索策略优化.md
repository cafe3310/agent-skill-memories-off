# 开发日志: Agent 探索逻辑与全文检索策略优化

## 1. 概述 (Overview)

针对 Agent 在初始化阶段「过度关注 meta.md 细节」以及在检索阶段「仅通过文件名判断内容是否存在」这两个痛点，本阶段对 Agent 的系统提示词 (Prompt) 和 Skill 核心规范进行了定向优化。

## 2. 优化重点 (Key Improvements)

### 2.1 探索顺序矫正 (Orientation Sequence)
- **强制先看工具**: 在 `prompts/个人助手.md` 和 `SKILL.md` 中均增加了明确指令：新任务开始时，**严禁**直接读取大型 `meta.md`。Agent **必须**首先运行 `memocli --help` 或查看 `SKILL.md` 的工具说明，在掌握「如何操作」后再去了解「规范细节」。
- **目的**: 避免 Agent 因不熟悉 `memocli` 子命令而导致的工具调用错误或行为偏差。

### 2.2 全文检索强制化 (Content-Grep First)
- **不再依赖文件名**: 明确规定文件名检索不再作为唯一判定标准。
- **强制 Grep 规则**: 当询问具体知识点、观点、方法论（如“评测怎么做”）时，Agent **必须**首先对 `entities/` 目录执行全文检索 (`grep_search` 或 `grep -C 2`)。
- **目的**: 确保 Agent 能发现隐藏在实体内容中的碎片化知识，提高检索的深度和准确性。

## 3. 修改文件 (Modified Files)

- `prompts/个人助手.md`: 重构了「工具使用要求」章节。
- `memories-off-skill/SKILL.md`: 在「约束与原则」中增加了「工具发现优先」和「全文检索优先」条款。

## 4. 验证与建议 (Validation & Tips)

- **建议**: 如果后续发现 Agent 依然表现出由于对工具不熟悉导致的失误，可考虑在 `memocli` 的输出中进一步增加对当前 Skill 规范文件的引用路径提示。
- **后续**: 配合 `TODO.md` 阶段 2，后续将进一步精简子命令的输出，以优化 Token 消耗。
