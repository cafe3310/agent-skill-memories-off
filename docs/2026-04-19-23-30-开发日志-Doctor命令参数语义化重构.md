# 开发日志: Doctor 命令参数语义化重构

## 1. 背景与问题
随着 `memocli doctor` 承担的自动修复责任越来越重，它的诊断规则也从最初的一两条增加到了多条（包括名称标准化、标题层级清洗、链接孤儿审计、旧版更新块修复等）。
之前各项规则的参数命名（如 `--normalize-name`, `--audit`）随意性较强，且容易与主命令或其他工具混淆，缺乏统一的“规则引擎”语义。

## 2. 改进内容
为了使参数层级更清晰、意图更明确，将所有 `doctor` 特有的诊断与修复触发器全部统一加上了 `--rule-` 前缀：

- `--normalize-name` -> `--rule-normalize-name`: 负责实体名、WikiLinks、关系目标名等核心元数据的标准化与修复。
- `--normalize-headers` -> `--rule-normalize-headers`: 负责清理历史遗留的凌乱 Markdown 标题，统一降维/升级至 H2 层级。
- `--audit` -> `--rule-audit-links`: 负责扫除孤立的空悬关系和死掉的 WikiLinks。
- `--fix-update-blocks` -> `--rule-fix-update-blocks`: 负责探测陈旧的 `UPDATE_BLOCK`，并基于文件修改时间等策略安全回退、补齐修复。

## 3. 影响范围
- 已同步更新 `commands/doctor.py` 中的解析器定义及 `help` 和 `example` 信息。
- 重新运行了安装脚本 `install.py`，更新了全局的 `memocli` 包装器。
- 未来如果继续在 `doctor` 内添加新的健康规则（例如别名去重等），也应严格遵守 `--rule-` 前缀规范。
