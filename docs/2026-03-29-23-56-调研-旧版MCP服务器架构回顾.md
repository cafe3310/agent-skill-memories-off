# 调研: 旧版 MCP 服务器架构回顾

## 1. 背景
`memories-off` 的初始版本（v1 和 v2）是作为一个标准的 MCP (Model Context Protocol) 服务器构建的。其设计初衷是为 Claude Desktop 等支持 MCP 协议的客户端提供一个结构化的、可编程的「长期记忆」接口。

## 2. 技术实现
- **语言与环境**: 使用 TypeScript 编写，运行在 Bun 环境下。
- **协议**: 严格遵循 MCP 规范，通过标准输入输出 (STDIO) 进行 JSON-RPC 通信。
- **工具定义**: 使用 `Zod` 定义了极其详尽的输入参数校验模式。
- **核心逻辑**: 
    - 所有的操作都是原子化的，如 `addEntityContent` 或 `replaceEntitySection`。
    - 系统会精确解析 Markdown 文件的标题结构 (TOC)，允许 Agent 定向读取特定章节以节省上下文。
    - 所有的元数据（Frontmatter）合并和冲突处理都在内存中通过逻辑计算完成。

## 3. 工具集能力
- **实体管理**: 增删改查、重命名、合并实体。
- **关系管理**: 在 Frontmatter 中显式建立 `relation as ...` 语义连接。
- **检索功能**: 提供基于元数据的过滤和正文 glob 匹配。
- **库维护**: 包含基础的备份功能和 meta.md 的编辑接口。

## 4. 局限性
- **环境要求**: 需要安装 Node.js/Bun 环境，且必须在 MCP 客户端中进行繁琐的配置。
- **交互成本**: Agent 在非 MCP 环境下（如纯 Shell 环境）很难直接调用这些工具，因为它们期望接收复杂的 JSON 对象作为参数。
- **路径依赖**: Agent 必须手动提供完整的库路径，无法根据当前工作目录自动识别。
