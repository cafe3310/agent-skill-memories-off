# 开发日志: 新增 memocli load 批量加载实体功能

## 1. 背景与目标
知识库在完成大范围检索后，Agent 往往需要精确加载多个目标实体的全文内容进行分析。原先只能使用系统的 `read_file` 逐个读取，效率低且受限于原生工具不支持解析别名或执行规范化的不足。

本次开发新增 `memocli load` 命令，提供一站式、原生的批量实体全文读取能力。

## 2. 主要修改内容

### 2.1 新增 `load.py` 脚本
- **位置**: `memories-off-skill/scripts/commands/load.py`
- **职能**: 接收逗号分隔的实体名列表，读取并返回它们包含 YAML Frontmatter 在内的完整 Markdown 内容。
- **参数**: `-n, --names`，支持以逗号分隔的实体名称或别名。
- **设计亮点**: 
  - 深度复用 `utility.runtime.ScriptBase` 和 `utility.schema_define.MetadataParser`，自动支持 `--path`, `--reason` 和标准错误输出机制。
  - 通过 `MetadataParser.normalize_name` 自动规范化输入的实体名，支持别名或非标准写法的容错查找。
  - 直接复用 `ScriptBase.add_result` 输出标准且 XML 转义安全的 `memocli-result` 格式，不再重复造轮子重写 XML。

### 2.2 更新操作定义
- 更新了 `操作定义/检索工具规格.md`。
- 将旧版的 `loadEntities` (依赖 `read_file`) 替换为正式的原生工具 `load`。

## 3. 使用方式示例
```bash
memocli load -p . -n "实体1,实体2,实体3"
```
系统将会按实体依次输出全文，缺失的文件也会在对应区块中给出 "文件不存在" 的提示。

## 4. 后续建议
基于当前的 `load` 命令，未来可以很容易地拓展实现 `load --section` 功能，用于仅加载指定实体的特定 H2 章节，进一步缩减上下文。
