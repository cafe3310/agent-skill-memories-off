# 设计: 多谓词关系注入与可重复参数增强

基于用户反馈，进一步增强 `memory-skill` 在创建和更新实体时的复杂织网能力。解决目前单个命令中无法同时定义多种不同谓词关系的限制。

## 1. 核心改进点

### 1.1 弃用混淆缩写
- **废弃**: `-rel` / `--relations`。
- **理由**: 该参数语义不明且与新的 `--add-rel-out` 功能重合，容易导致使用者混淆。

### 1.2 参数可重复化 (Repeatable Action)
- **特性**: 允许在同一命令中多次使用 `--add-rel-out` 和 `--add-rel-in`。
- **逻辑**: 
    - 每一个参数实例都代表一个独立的原子操作。
    - 脚本需遍历所有出现的参数实例，并按顺序依次执行关系建立。
- **示例**: 
  ```bash
  memocli create-entity -n "Alice" \
    --add-rel-out "member:TeamA,TeamB" \
    --add-rel-out "author:Doc1" \
    --add-rel-in "manager:BossX" \
    -r "初始化 Alice 及其复杂关联"
  ```

## 2. 影响范围分析

### 2.1 参数解析逻辑 (`schema_define.py` 不涉及，仅限具体脚本)
- 需要修改 `argparse` 的配置，将参数设置为 `action="append"`。这意味着解析后的结果将是一个列表（List of Strings），而非单一字符串。

### 2.2 业务逻辑脚本
- **`manage_relations.py`**: 需适配列表循环。
- **`create_entity.py`**: 需适配列表循环。
- **`append_update.py`**: 需适配列表循环。

### 2.3 语义健壮性
- 多次 `add-rel-out` 作用于同一个谓词时，逻辑应保持「追加」而非「覆盖」。
  - 例：`--add-rel-out "friend:B" --add-rel-out "friend:C"` 应生成 `friend: B, C`。

## 3. 实施路径

1.  **升级 `manage_relations.py`**: 作为核心管理工具首先适配可重复参数。
2.  **升级 `create_entity.py`**: 彻底移除 `-rel`，适配可重复参数。
3.  **升级 `append_update.py`**: 适配可重复参数。
4.  **重新安装 `memocli`**: 更新帮助信息中的示例。

## 4. 预期效果
- 极大提升 Agent 在复杂任务中的单步执行效率。
- 使得 `memocli` 能够胜任高密度的知识图谱初始化工作。
