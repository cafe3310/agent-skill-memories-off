# 开发日志: 子命令错误提示增强与基类重构 (Subcommand-Error-Enhance)

## 1. 概述 (Overview)

为了显著降低 Agent 在遇到参数错误时的纠错成本，本任务对 `schema_define.py` 进行了重构，引入了统一的脚本基类和定制化的参数解析器。

### 1.1 来源文档与背景
- **设计文档**: `docs/2026-04-07-08-36-设计-子命令错误提示增强与多语言优化.md`
- **核心痛点**: 
    - 子命令报错为英文。
    - 报错后不显示用法 (Usage)，导致 Agent 需额外回合获取帮助。
    - 报错信息未包裹在 XML 报告中，不符合 Agent-First 协议。

---

## 2. 核心改进 (Key Improvements)

### 2.1 引入 CustomArgumentParser
- **翻译映射**: 实现了对 `argparse` 常用错误信息的实时翻译（如 `the following arguments are required` -> `缺少必要参数`）。
- **自动引导**: 覆写了 `error` 方法，在报错时自动调用 `print_help()`。
- **XML 协议**: 在报错输出末尾强制追加 `<{action}_report success="false">` 块，提供明确的 `instruction`。

### 2.2 脚本基类 ScriptBase
- **统一生命周期**: 提供 `setup()`, `run()`, `finalize()` 标准流程。
- **结构化输出**: 内置 `add_element`, `log`, `error` 方法，自动处理 XML 转义与标签包裹。
- **协议兼容**: 自动处理 `--memo-cli-info` 和 `--memo-cli-call`。

### 2.3 典型应用重构
- **stats.py**: 已完成重构，成为首个基于 `ScriptBase` 的子命令。其输出现在完全符合「模型优先」的 XML 包裹规范。
