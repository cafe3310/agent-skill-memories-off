# 设计文档：memocli 枚举参数优化方案 (Enriched-Enum-Help)

## 1. 背景 (Problem Statement)

目前一些子命令（如 `commit`）包含带有 `choices` 的枚举参数。
`argparse` 默认虽然能列出这些 choices，但缺乏对每个选项具体语义的解释。
这导致 Agent 在选择 `action` 时可能犹豫不决（例如 `fix` vs `edit`）。

---

## 2. 核心目标 (Core Objectives)

1.  **语义显例化**：在帮助系统中直接列出每个选项及其对应的操作含义。
2.  **易读性**：通过合理的缩进和格式化，使枚举说明清晰可见。
3.  **协议一致性**：通过 `ScriptBase` 统一输出，并由 `install.py` 转发至全局帮助。

---

## 3. 实施方案 (Implementation Plan)

### 3.1 `ScriptBase` 增强 (`schema_define.py`)
在 `ScriptBase` 的 `__init__` 中增加 `enum_details` 参数，用于定义特定参数的详细选项说明。

- **动作**：修改 `_check_info()` 协议，使其在输出 `--memo-cli-info` 时同时包含枚举细节。
- **动态描述追加**：在 `_init_parser` 中，自动将枚举详情追加到 `self.parser.description` 中。

### 3.2 脚本更新 (`scripts/*.py`)
以 `commit.py` 为例，更新其定义：
```python
        super().__init__(
            action_name="commit",
            description="知识库变更的标准化 Git 提交工具（Audit Log）。",
            enum_details={
                "action": {
                    "create": "创建实体",
                    "edit": "修改内容",
                    "rename": "重命名文件",
                    "delete": "移动到垃圾箱",
                    "fix": "审计修复",
                    "init": "初始化仓库",
                    "merge": "合并实体",
                    "consolidate": "整合缓冲更新"
                }
            }
        )
```

### 3.3 安装器增强 (`install.py`)
更新 `create_memocli()`，使 `memocli --help` 能够解析并展示由 `--memo-cli-info` 输出的 `Enum Details:` 部分。

**预期输出效果**：
```text
  commit               - 知识库变更的标准化 Git 提交工具（Audit Log）。
    Example: memocli commit --path . --action edit --target "实体名" --reason "理由"
    Enum (action):
      - create      : 创建实体
      - edit        : 修改内容
      - rename      : 重命名文件
      ...
```

---

## 4. 验证标准 (Validation)

1.  运行 `memocli commit --help`：应看到详细的枚举说明。
2.  运行 `memocli --help`：全局子命令列表下方应包含这些枚举详情。
