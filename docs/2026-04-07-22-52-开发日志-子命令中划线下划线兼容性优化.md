# 开发日志: 子命令中划线与下划线兼容性优化 (Subcommand-Naming-Cohesion)

## 1. 概述 (Overview)

用户和 Agent 在调用 `memocli` 时经常混淆中划线 (`-`) 和下划线 (`_`)。本任务旨在通过包装器层的逻辑映射，实现对两种命名风格的完全兼容，并统一文档中的展示标准。

### 1.1 背景与目标
- **目标**: 使 `memocli get-relations` 和 `memocli get_relations` 具有等效行为。
- **展示标准**: 在帮助信息和示例中，优先推荐并展示中划线风格。

---

## 2. 核心改进 (Key Improvements)

### 2.1 包装器自动映射 (Shell Mapping)
- **修改文件**: `memories-off-skill/scripts/install.py` (内联 Bash 逻辑)
- **实现**: 在执行子命令前，通过 `tr '-' '_'` 将 `$ACTION` 中的中划线统一转换为下划线，以匹配底层 Python 脚本文件名。

### 2.2 自动发现机制升级
- **改进**: `install.py` 在安装阶段扫描 `scripts/*.py` 时，会自动将文件名中的下划线替换为中划线进行展示。
- **效果**: `memocli --help` 的输出列表现在整齐地以中划线形式呈现（如 `get-relations`, `audit-fix`）。

### 2.3 子命令内部 Usage 兼容
- **细节**: 虽然底层文件是 `get_relations.py`，但在其 `argparse` 的 `prog` 定义中，已通过 `replace("_", "-")` 确保了 `--help` 输出中的 usage 引导也采用中划线风格。
