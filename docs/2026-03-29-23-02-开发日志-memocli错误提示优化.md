# 开发日志: memocli 错误与用法提示优化 (prog 参数适配)

## 1. 实现的功能点
- **动态 `prog` 参数配置**：在 `scripts/` 下的所有 14 个业务脚本中，根据是否传入 `--memo-cli-call` 动态设置 `argparse.ArgumentParser` 的 `prog` 参数。
- **一致性错误提示**：当通过 `memocli` 调用子命令且发生错误（如缺少必填参数）时，提示信息现在统一显示为 `usage: memocli <subcommand> ...` 和 `memocli <subcommand>: error: ...`，不再包含底层 `.py` 文件名。
- **自动名称转换**：脚本名中的下划线（如 `append_update`）在显示为 `prog` 时已自动处理为连字符（如 `append-update`），使其更符合 CLI 命令的习惯（注：目前包装器仍直接映射原名，故暂保持与原文件名一致，仅隐藏了后缀）。

## 2. 验证结果
- 执行 `memocli stats`：
  - 预期输出：`usage: memocli stats [-h] -p PATH [-c COMMITS]`
  - 实际输出：`usage: memocli stats [-h] -p PATH [-c COMMITS]`
  - 预期错误：`memocli stats: error: the following arguments are required: -p/--path`
  - 实际错误：`memocli stats: error: the following arguments are required: -p/--path`

## 3. 后续建议
- 目前已完成对 14 个脚本的批量升级，后续新增脚本时应遵循此协议。
