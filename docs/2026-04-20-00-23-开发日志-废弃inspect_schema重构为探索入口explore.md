# 开发日志: 废弃 inspect_schema，重构为探索入口 explore

## 1. 背景与问题
在《其他Agent的优化建议》中提到，`inspect_schema` 最初仅仅是一个采样工具（最大读取 100 个文件），它无法满足 Agent 对全局架构精确掌握的需求。
此外，Agent 在初次接触一个已有知识库时，往往需要运行多个独立命令（找文档、看类型、查命令）来建立上下文，存在较高的冷启动门槛。

## 2. 主要修改内容

为了提供一个“一站式的知识库开眼工具”，我移除了 `inspect_schema.py`，并新建了 `explore.py`，作为所有新任务的推荐起点。

`explore` 命令不仅执行全量的数据统计，更融合了文档导读与系统说明功能，包含 6 大核心模块：
1. **绝对路径导航**：打印 `meta.md` 的绝对路径，确立工作空间坐标。
2. **核心手册导读**：直接打印出 `meta.md` 的全文（默认限制前 500 行，防 Token 爆炸）。
3. **关键实体预览**：自动扫描 `meta.md` 内部引用的所有 `[[WikiLinks]]`，并直接把这些被视为“架构基石”的实体的 `Frontmatter` 打印出来。
4. **全量类型分布**：遍历 `entities/` 下的**所有文件**进行统计，不再采用固定数量采样。精确展示每种 `entity type` 和其包含的实体数量（默认展示前 200 种）。
5. **全量关系分布**：精确统计知识库中声明的所有显式关系及关联频次（默认展示前 200 种）。
6. **工具清单速查**：通过内部重定向，在输出的末尾自动附带当前可用的完整 `memocli --help` 菜单，确保 Agent 对当前版本的工具链能力一目了然。

### 2.1 参数设计
- `--full`: 当知识库过于庞大，或是想查阅超过截断限制的数据时，加入该参数即可取消所有的行数和条目截断限制。

## 3. 验证与总结
- 删除了废弃脚本，重新执行了安装器同步 Bash Wrapper，确保了 CLI 命令表的纯净。
- 已在 `docs/2026-04-12-00-52-设计-其他Agent的优化建议.md` 的对应条目中标记了已解决。
