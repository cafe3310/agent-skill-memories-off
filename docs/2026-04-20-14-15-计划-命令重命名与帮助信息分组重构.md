# 计划: memocli 命令重命名与帮助信息分组重构

## 1. 目标
随着 `memocli` 命令数量的增加，现有的扁平化帮助菜单难以阅读。本次重构旨在：
1. 对部分命令进行更符合直觉的重命名（强调单复数及动作目标）。
2. 在底层的 `ScriptBase` 引入分组概念，使 `memocli --help` 能够按逻辑分组展示可用命令。

## 2. 命令变更映射与分组

### 快速加载上下文
- `explore` -> `explore`

### 写知识
- `create-entity` -> `create-entity`
- `append-update` -> `append-update`

### 检索和读知识
- `search` -> `search-entities`
- `load` -> `load-entities`
- `get-relations` -> `get-relations`

### 编辑
- `rename` -> `rename-entity`
- `rename-type` -> `rename-type`
- `merge-entities` -> `merge-entities`
- `merge-types` -> `merge-types`
- `manage-relations` -> `manage-relations`

### 对知识库操作
- `graph` -> `graph`
- `stats` -> `stats`
- `commit` -> `commit`
- `doctor` -> `doctor`
- `init` -> `init`
- `consolidate-updates` -> `squash-updates`

## 3. 执行步骤

### 阶段一：底层架构修改
1. **修改 `utility/runtime.py`**:
   - 在 `ScriptBase.__init__` 中新增 `group_name: str` 参数，默认为 `"未分类"`。
   - 在 `_check_info` 方法中输出 `Group: {self.group_name}` 以供 `install.py` 解析。
2. **修改 `install.py`**:
   - 调整子命令探测逻辑，解析新增加的 `Group` 字段。
   - 维护一个按照分组组织的命令字典。
   - 在生成的 bash wrapper 的 `help` 分支中，按分组迭代并打印标题和命令信息。

### 阶段二：命令重命名与元数据更新
1. 针对发生名称变更的命令，重命名其对应的 `.py` 脚本文件：
   - `search.py` -> `search_entities.py`
   - `load.py` -> `load_entities.py`
   - `rename.py` -> `rename_entity.py`
   - `consolidate_updates.py` -> `squash_updates.py`
2. 修改所有命令脚本：
   - 将 `action_name` 变更为新的名称（使用中划线或下划线，框架会自动处理）。
   - 在 `super().__init__(...)` 中注入对应的 `group_name`。

### 阶段三：文档与规范同步
全局替换以下文档中过时的命令名称：
- `README.md`
- `SKILL.md`
- `操作定义/观察工具规格.md`
- `操作定义/检索工具规格.md`
- `操作定义/编辑工具规格.md`
- `操作定义/管理工具规格.md`
- `prompts/通用-个人助手.md`

## 4. 验证
重新运行 `python scripts/install.py`，然后执行 `memocli --help` 验证分组显示是否正确，并验证帮助菜单排版。
