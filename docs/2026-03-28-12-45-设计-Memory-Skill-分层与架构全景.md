# 设计文档: Memory-Skill 分层与架构全景

本文档系统性地梳理了 `memory-skill` 的整体架构分层、核心定义及动作分类，确保 Agent 在开发和使用过程中具备完整且不重不漏的全局视角。

---

## 1. 核心架构分层 (Core Layers)

我们将系统划分为四个互不重叠、职责明确的抽象层：

### L1: 仓库定义层 (Repository/Library Layer)
*职责: 定义多个文件如何作为一个逻辑整体（Library）共存。*
- **顶层结构**: `meta.md`, `entities/`, `trash/`, `backups/`, `journeys/`。
- **手册规范**: `meta.md` 作为全局描述、Schema 定义和规则手册。
- **关联文档**: `memory-skill/知识库结构定义/`。

### L2: 文件定义层 (Document/File Layer)
*职责: 定义单个 Markdown 文件的内部物理结构与逻辑解析逻辑。*
- **物理结构**: Front Matter (YAML 元数据) + Body (Markdown 正文)。
- **逻辑模型**: `Document` = `Metadata` + `TOC` + `Sections` + `Content`。
- **安全协议**: `ContentLocator` 的“唯一性验证”机制与标准化清洗规则。
- **关联文档**: `memory-skill/文档结构定义/`。

### L3: 关联定义层 (Relation Layer)
*职责: 定义实体间的拓扑连接，构建知识图谱。*
- **显式关系**: 存储在 Front Matter 中的 `relation as [predicate]`。
- **隐式链接**: 存储在 Body 中的 WikiLinks `[[ThingName]]`。
- **同步协议**: 实体重命名时的全局链接更新逻辑。
- **关联文档**: `memory-skill/关联定义/`。

### L4: 动作定义层 (Action/Tool Layer)
*职责: 定义 Agent 可调用的原子化工具规格，实现对 L1-L3 层的操作。*
- **关联文档**: `memory-skill/操作定义/`。

---

## 2. 动作分类 (Action Categories)

为了优化 Agent 的调用逻辑，动作被划分为以下四大功能类：

| 分类 | 核心目的 | 典型工具示例 |
| :--- | :--- | :--- |
| **库观察 (Observation)** | 获取全局上下文、Schema、统计信息及规则。 | `loadManual`, `getLibraryStats` |
| **检索与发现 (Discovery)** | 在海量实体中精确定位目标实体的名称。 | `findEntitiesByMetadata`, `searchEntitiesByContent` |
| **读取与解析 (Reading)** | 获取实体的详细层级结构和具体文本内容。 | `loadEntitiesToc`, `loadEntitiesSections` |
| **修改与组织 (Mutation)** | 改变文件系统状态及图谱结构（含审计原因）。 | `addContentToSection`, `replaceSection`, `renameEntity` |

---

## 3. 审计与元协议 (Audit & Meta-Protocols)

### 3.1 审计日志 (Audit Logs)
- 所有 `Mutation` 类操作必须在 `journeys/` 目录下留下变更痕迹。
- **关联文档**: `memory-skill/审计与日志定义/journey-specification.md`。

### 3.2 XML 报告协议 (XML Reporting)
- 工具返回统一的 XML 格式，包含 `<status>`, `<data>`, `<error>` 及操作前后的 `diff` 摘要。

### 3.3 唯一性匹配协议 (Uniqueness)
- 任何基于内容的编辑必须满足在搜索范围内有且仅有一个匹配项。
