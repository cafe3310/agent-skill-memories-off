# 设计: 关系管理增强与多值支持方案 (方案 A)

基于用户反馈，针对 `memory-skill` 的关系管理进行重构，采用 `--rel-out` 和 `--rel-in` 命名规范，解决单值覆盖问题并提升创建/更新效率。

## 1. 核心定义

### 1.1 存储逻辑：多值支持 (Multi-value Support)
- **格式**: 同一谓词下的多个目标以「逗号分隔的字符串」存储。
- **YAML 格式**: `relation as [谓词]: 目标1, 目标2`。
- **逻辑**: 添加时检查去重，删除时局部移除。

### 1.2 操作逻辑：Inbound 与 Outbound
所有涉及实体创建或更新的工具统一支持以下参数：

| 参数 | 语义 | 逻辑方向 | 修改位置 |
| :--- | :--- | :--- | :--- |
| `--rel-out "p:t1,t2"` | 出站 (Outbound) | Current -> [p] -> Target | 修改 **当前操作实体** |
| `--rel-in "p:s1,s2"` | 入站 (Inbound) | Source -> [p] -> Current | 修改 **来源实体** |

## 2. 实施路径

### 2.1 底层支持 (`schema_define.py`)
在 `MetadataParser` 中新增：
- `add_relation(metadata, predicate, target)`: 将目标安全地添加到元数据中，支持逗号分隔。
- `remove_relation(metadata, predicate, target)`: 从逗号分隔的列表中移除特定目标。

### 2.2 脚本升级
1.  **`manage_relations.py`**: 
    - 废弃 `--add` / `--remove`，采用 `--rel-out` 和 `--rel-in`。
    - 修复覆盖 Bug。
2.  **`create_entity.py`**: 
    - 允许在创建时通过 `--rel-out` 定义自己的关系。
    - 允许通过 `--rel-in` 自动修改已有实体的关系网，将新实体“挂载”上去。
3.  **`append_update.py`**: 
    - 支持在追加内容时，同步更新元数据关系。

## 3. 示例场景

**场景：创建项目 A，并建立复杂关联**
```bash
memocli create-entity -n "ProjectA" -t "Project" \
  --rel-out "member:Alice,Bob" \
  --rel-in "part of:CompanyX" \
  -r "初始化项目 A 及其关联"
```
**结果**:
1. 创建 `ProjectA.md`，其中包含 `relation as member: Alice, Bob`。
2. 修改 `CompanyX.md`，在其 `relation as part of` 中追加 `ProjectA`。
