# 2026-03-28-12-15-需求与设计-Memory-Skill-结构规划

用户要求在根目录创建一个 `memory-skill` 目录，作为一个完整的 Agent Skill 来实现类似 v2 的功能。

## 1. 背景与目标 (Background & Goals)

本项目已将旧代码归档至 `legacy/`，目前的战略是先以 Agent Skill 风格构建 v2 版本。`memory-skill` 将作为承载这一功能的逻辑实体，定义 LLM 如何与本地知识库进行交互。

## 2. 核心概念与结构 (Core Concepts & Structure)

基于对 `legacy/src/v2` 和 `docs/` 的分析，我们将 `memory-skill` 划分为以下子模块：

### 2.1 整体介绍 (`skill.md`)
- 定义 Skill 的名称、版本、主要能力、使用场景及约束。
- 阐述 Agent 与该 Skill 交互的基本流程。

### 2.2 知识库结构定义 (`知识库结构定义/`)
- **`library-layout.md`**: 定义 `meta.md`, `entities/`, `trash/`, `backups/`, `journeys/` 等顶层目录的作用和规则。
- **`meta-specification.md`**: 定义 `meta.md` 的内部结构（Manual）。

### 2.3 文档结构定义 (`文档结构定义/`)
- **`entity-structure.md`**: 定义实体文件的 Markdown 规范（Frontmatter + Body）。
- **`metadata-schema.md`**: 定义标准的元数据字段（`entity type`, `date modified`, `aliases` 等）。
- **`section-logic.md`**: 定义章节（Section）、标题（Heading）和目录（TOC）的逻辑关系。

### 2.4 关联定义 (`关联定义/`)
- **`relation-types.md`**: 定义实体间显式关系的表示方式（如 `relation as [type]: [target]`）。
- **`link-logic.md`**: 定义隐式链接（WikiLinks `[[...]]`）的处理逻辑。

### 2.5 操作定义 (`操作定义/`)
- **`observation-tools.md`**: 定义库观察操作（stats, load manual）。
- **`retrieval-tools.md`**: 定义检索操作（search by name/metadata/content）。
- **`editing-tools.md`**: 定义编辑操作（add/replace section）。
- **`management-tools.md`**: 定义实体管理操作（create, rename, merge, trash）。

## 3. 遗漏概念与补充建议 (Missing Concepts & Suggestions)

参考 `v2` 源码，我们还需考虑以下“元操作”或“辅助协议”：
- **查询语法 (Query Syntax)**: 定义 `FileGlobPattern` 和 `SubstringQuery` 的精确行为。
- **报告格式 (Reporting)**: 统一 XML Report 的结构，确保 Agent 能解析结果。
- **审计与日志 (Reasoning)**: 强制要求提供 `reason` 参数以便记录。
- **回收机制 (Garbage Collection)**: 处理断裂的关系和孤立的实体。
