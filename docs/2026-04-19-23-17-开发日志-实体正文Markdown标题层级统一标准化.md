# 开发日志: 实体正文 Markdown 标题层级统一标准化

## 1. 背景与问题
在过往的协同编辑中，用户或 Agent 往往随意使用不同层级的 Markdown 标题（例如 `#`, `###`, `####` 等），这导致实体内部缺乏统一的文档结构大纲（TOC）。
为了支持未来“按章节检索和按需加载”等高级能力的演进，我们需要建立一个全局的排版共识：
- 实体正文的第一行固定为 H1 (`# 实体名称`)。
- 所有属于实体的**业务正文块中的标题，必须统一拍平并标准化为 H2 (`## 章节名`)**。

## 2. 主要修改内容

### 2.1 底层抽象：`MetadataParser.normalize_headers`
在 `utility/schema_define.py` 中实现了安全的 Markdown 标题层级清洗逻辑：
- 按行扫描，将任何满足 `^#+\s+(.*)` 的行转写为 `## \1`。
- **安全过滤**：通过维持一个状态标志 `in_code_block`，跳过代码块 (```) 内的所有内容，防止误伤含有注释的脚本片段。

### 2.2 拦截新增写入：`UpdateBlockManager`
我们在之前重构的 `UpdateBlockManager.create_block()` 方法中加入了对 `normalize_headers` 的调用。
这意味着现在所有通过 `create-entity --content` 或是 `append-update --content` 追加的新内容，都会在写入磁盘前，自动经历 WikiLink 标准化和 Markdown Header 的 H2 标准化。

### 2.3 全局纠错：`memocli doctor --normalize-name`
在 `commands/doctor.py` 的全局标准化通道中，新增了针对存量实体的标题诊断和修复逻辑：
- 它会安全地剥离出文件头部的原始 H1 `# 实体名`，然后将剩余正文扔进清洗逻辑。
- 如果检测到旧文件使用了不规范的标题层级，在带有 `--fix` 参数时会自动覆盖文件并给出提示。

## 3. 验证与总结
所有修改均在临时初始化的测试库中验证通过。新传入的 `### Deep Section` 会被正确拍平为 `## Deep Section`，而代码块内诸如 `# This is a comment` 的结构则毫发无损。这为我们后续开发更加轻量的局部加载工具（`load --section`）做好了数据准备。
