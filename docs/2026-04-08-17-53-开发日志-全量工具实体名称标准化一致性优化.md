# 开发日志：全量工具实体名称标准化一致性优化 (Entity-Naming-Consistency)

## 1. 概述 (Overview)
为了解决实体名称在创建、查询、编辑和搜索环节中转义逻辑不一致导致的“逻辑断裂”问题，本任务对全量工具集进行了审计与修正，确立了“输入宽容、存储严格”的统一处理原则。

## 2. 核心改进 (Actions)

### 2.1 统一名称预处理 (Normalization Injection)
- **修改工具**: `append-update`, `get-relations`, `manage-relations`, `merge-entities`。
- **改进内容**: 在所有涉及“实体名称”输入的地方，第一时间调用 `MetadataParser.normalize_name()`。
- **效果**: 用户或 Agent 输入 `A:B` 或 `A/B`，工具链会自动映射到物理存储格式 `A-B.md`，彻底消除了“文件不存在”的误报。

### 2.2 搜索体验优化 (Smart Search Mapping)
- **修改工具**: `search.py`。
- **逻辑**: 
    - 针对 `--name` (名称搜索) 模式，自动将搜索词标准化。
    - 例如：搜 `测试:名称` 会自动转换为搜 `测试-名称`，从而匹配到标准化的物理文件名。
    - 保持了正文搜索的原始灵活性。

### 2.3 代码健壮性 (Robustness)
- 修复了 `search.py` 中因变量定义顺序导致的 `NameError`。
- 修复了 `search.py` 中路径解析逻辑在处理包含中划线的路径时的 Bug。

## 3. 验证记录 (Validation)
- [x] **创建**: `create-entity -n "测试:名"` -> 生成 `测试-名.md` (成功)。
- [x] **查询**: `get-relations --entity "测试:名"` -> 成功定位并分析 `测试-名.md`。
- [x] **搜索**: `search "测试:名" --name` -> 自动映射并匹配到 `测试-名.md` (成功)。
