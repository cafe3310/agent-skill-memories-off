# 开发日志: memocli 安装与功能转发机制实现

## 1. 实现的功能点
- **自动化安装脚本 (`install.sh`)**：支持自动检测 `/usr/local/bin` 或 `~/.local/bin` 作为安装路径，并将项目脚本目录的绝对路径注入到包装器中。
- **命令行包装器 (`memocli`)**：
    - 支持 `memocli <action> <args>` 格式。
    - 自动映射 `action` 到对应的 `.py` 脚本。
    - **路径智能识别**：在包含 `meta.md` 的目录下执行时，若缺省 `--path` 参数，自动注入 `--path .`。
- **设计文档**：撰写了 `docs/2026-03-29-22-01-设计-memocli-安装机制.md`。

## 2. 代码修改
- **新增**：`memories-off-skill/scripts/install.sh`。
- **更新**：`docs/TODO.md`（标记任务进度）。

## 3. 遇到的问题与解决
- **PATH 问题**：在非登录 Shell 或新安装环境下，`~/.local/bin` 可能不在 `PATH` 中。已在 `install.sh` 中添加警告提醒用户。
- **参数转发**：使用了 `shift` 和 `"$@"` 确保原始参数能被透传给 Python 脚本。

## 4. 后续建议
- **自动补全**：目前 `memocli` 不带参数运行会显示可用列表，未来可以进一步提供 Bash/Zsh 的自动补全脚本。
- **标准化输出**：随着 `memocli` 的使用，应开始优化各脚本的 XML 输出，以支持更高效的 Token 压缩。
