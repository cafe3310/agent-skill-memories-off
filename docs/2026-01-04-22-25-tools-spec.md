# 工具规格

本文档遵循“知识检索”和“知识加载”的核心概念，对所有工具进行重新分类，以更好地反映其在知识生命周期中的作用。

格式为：`name(arg1=描述1, arg2=描述2, ...) -> 输出描述`

## 知识创建 (Knowledge Creation)

- createEntities(libraryName=知识库名称, entity=要创建的文档或文档列表, reason?=该调用的简要目的) -> XML 格式的报告，包含成功和失败的实体列表。

## 知识检索 (Knowledge Retrieval)

- findEntitiesByName(libraryName=知识库名称, entityNameGlobs=用于匹配实体名称的 glob 模式或模式列表, reason?=该调用的简要目的) -> XML 格式的报告，包含所有匹配到的实体名称列表。
- summarizeEntityTypes(libraryName=知识库名称, reason?=该调用的简要目的) -> XML 格式的报告，列出所有唯一的实体类型及其数量。
- findEntitiesByMetadata(libraryName=知识库名称, metadataQuery=用于在实体元数据中搜索的键值对, reason?=该调用的简要目的) -> XML 格式的报告，返回匹配元数据查询的实体列表。
- findRelations(libraryName=知识库名称, toEntity?=关系指向的目标实体, relationType?=关系类型) -> (尚未完全实现) 查找知识库中的关系。
- findEntitiesByContent(libraryName=知识库名称, contentGlob=用于在实体正文中搜索的 glob 模式, reason?=该调用的简要目的) -> XML 格式的报告，返回在实体正文中匹配 glob 模式的行及其章节标题。
- findEntitiesGlobally(libraryName=知识库名称, pattern=用于在任何地方搜索的 glob 模式) -> (尚未实现) 在文件名、元数据和正文中进行全局 glob 模式匹配。

## 知识加载 (Knowledge Loading)

- loadEntities(libraryName=知识库名称, entityNames=要读取的实体名称列表, reason?=该调用的简要目的) -> XML 格式的报告，加载一个或多个实体的完整内容。
- loadEntitiesToc(libraryName=知识库名称, entityNames=要获取目录的实体名称或名称列表, reason?=该调用的简要目的) -> XML 格式的报告，加载一个或多个实体的目录（TOC）。
- loadEntitiesSections(libraryName=知识库名称, entityNames=要读取章节的实体名称或名称列表, sectionGlobs=用于匹配章节标题的 glob 模式或模式列表, reason?=该调用的简要目的) -> XML 格式的报告，加载一个或多个实体中特定章节的内容。
- loadManual(libraryName=知识库名称, reason?=该调用的简要目的) -> XML 格式的报告，加载 `meta.md` 文件的完整内容。

## 知识编辑 (Knowledge Editing)

- addContentToSection(libraryName=知识库名称, entityName=要追加内容的实体名称, inSection=要追加内容的目标章节标题, newContent=要追加的 Markdown 内容, reason?=该调用的简要目的) -> XML 格式的报告，指示操作成功或失败。
- deleteContentFromSection(libraryName=知识库名称, entityName=要删除内容的实体名称, inSection=要删除内容的目标章节标题, contentToDelete=要精确删除的 Markdown 内容, reason?=该调用的简要目的) -> XML 格式的报告，指示操作成功或失败。
- replaceSection(libraryName=知识库名称, entityName=要替换章节的实体名称, oldHeading=要替换的旧章节标题, newHeading=新章节标题, newBodyContent=新章节的完整正文内容, reason?=该调用的简要目的) -> XML 格式的报告，重写整个章节。
- editManual(libraryName=知识库名称, oldLines=要被替换的整行旧内容块, newLines=要换成的整行新内容块, reason?=该调用的简要目的) -> XML 格式的报告，包含编辑后 meta.md 文件的完整内容。

## 知识连接 (Knowledge Connection)

- createRelations(libraryName=知识库名称, relations=格式为“source, verb, target”的关系字符串或字符串数组, reason?=该调用的简要目的) -> XML 格式的报告，包含成功和失败的关系列表。
- deleteRelations(libraryName=知识库名称, relations=格式为“source, verb, target”的关系字符串或字符串数组, reason?=该调用的简要目的) -> XML 格式的报告，包含成功和失败的关系列表。

## 知识组织 (Knowledge Organization)

- trashEntities(libraryName=知识库名称, entityNames=要删除的实体名称或名称列表, reason?=该调用的简要目的) -> XML 格式的报告，包含成功和失败的实体列表。
- renameEntity(libraryName=知识库名称, oldName=实体的旧名称, newName=实体的新名称, reason?=该调用的简要目的) -> XML 格式的报告，指示操作成功或失败，并自动更新所有指向它的链接。
- mergeEntities(libraryName=知识库名称, sourceNames=要合并的源实体名称列表, targetName=合并的目标实体名称, reason?=该调用的简要目的) -> XML 格式的报告，将多个源实体合并到一个目标实体，并返回合并后的内容。
- garbageCollectRelations(libraryName=知识库名称, dryRun?=是否为演习模式, reason?=该调用的简要目的) -> (尚未实现) 清理指向不存在实体的“断裂链接”。

## 备份管理 (Backup Management)

- backupLibrary(libraryName=要备份的知识库的名称) -> 确认信息，包含备份文件的路径和大小。
