# 工具规格

本文档遵循“领域驱动”和“操作对象”的原则，对所有工具进行重新分类，以便 LLM 根据当前任务的意图快速定位所需工具。

格式为：
```
- $name($argname-$argdesc: $argtype, ...)
  $描述
  $结果描述
  开发 ✅
  测试 ✅
```

# MCP Server 综述
`mcp-server-memories-off` 是一套用于长期记忆管理的本地优先知识库系统。它通过一系列原子化工具，赋予 LLM 对结构化 Markdown 文件的增删改查（CRUD）能力。

核心交互概念：
1.  **XML Report**: 所有工具均返回 XML 格式的结构化报告，旨在让 LLM 更清晰地解析执行结果（成功或失败的数据），而非供人类直接阅读。
2.  **Reason**: 几乎所有工具都包含一个 `reason` 参数。LLM 必须在此说明调用该工具的简要目的（例如“为了更新用户的生日信息”），这有助于审计和调试。
3.  **Manual (`meta.md`)**: 每个知识库都有一个核心元数据文件。使用 `loadManual` 可以获取该库的全局描述、Schema 定义和用户偏好，是 LLM 理解当前知识库上下文的首要入口。

核心查询模式：
1.  **FileGlobPattern**: 用于匹配文件名的模式。仅支持基础通配符 `*` (任意字符序列) 和 `?` (单字符)，**不支持**目录递归 (`**`)。
2.  **TextGlobPattern**: 用于在文件正文中搜索内容的简易模式。仅支持 `*` 和 `?`，不支持复杂的正则语法。
3.  **SubstringQuery**: 专门用于匹配章节标题。不区分大小写，只要包含该子字符串即视为匹配（例如 "history" 可命中 "2. History of Project"）。
4.  **MetadataQuery**: 用于精确查找 Frontmatter 元数据的键值对对象（如 `{ "tags": "wip" }`）。
5.  **SectionHeading**: 指章节标题文本，**不包含** Markdown 的 `#` 符号（例如应为 `History` 而非 `## History`）。必须**精确匹配**。

# 实现策略

为兼顾性能与安全，系统采用**Shell 加速检索 + 文件系统原子读写**的双层架构。

## 1. ⚡ Shell 加速 (Shell Acceleration)
对于检索（Retrieval）、过滤和统计类操作，优先使用原生 Shell 命令（如 `ls`, `grep`, `ripgrep`, `find`）进行快速筛选。
*   **优势**: 在处理大量文本文件时速度极快，远超 Node.js 层面的循环读取。
*   **场景**: `searchEntitiesByName` (ls), `searchEntitiesByContent` (grep/rg), `findEntitiesByMetadata` (grep Frontmatter), `getLibraryStats` (wc/grep)。

## 2. ✏️ 原子读写 (Atomic FS Operations)
对于具体的读取（Read）和修改（Edit）操作，利用项目现有的核心编辑器模块 (`src/entities/editor/`)，基于 `fs` 模块进行精确控制。
*   **流程**: 检索阶段返回文件路径列表 -> Node.js 逐个读取文件内容 -> 在内存中解析/修改 AST 或文本 -> 写入文件。
*   **关键模块与方法**:
    *   **File Ops** (`file-ops.ts`): `readFileContent`, `writeFileContent` (基础读写，含路径解析与存在性检查)。
    *   **Document Parsing** (`document.ts`): `parseDocument` (将 Markdown 解析为 Document 结构，含 FrontMatter, TOC, Sections)。
    *   **Content Editing** (`editing.ts`): `replaceSection`, `addContentToThing`, `replaceInHeading` (实现基于 locator 的精确文本修改)。
    *   **Locator** (`locator.ts`, `toc.ts`): `matchHeadingInToc`, `resolveContentLocator` (确保操作目标的唯一性和准确性)。
*   **原则**:
    *   **Verify-before-Write**: 在写入前必须验证文件状态（如检查章节是否存在、内容是否唯一）。
    *   **Fail-Fast**: 一旦前置检查失败（如文件被锁定或格式错误），立即中止并报错。

# 工具分类

## 1. 库观察 (Library Observation)
*用于了解知识库的全局状态、元数据结构和统计信息，通常在任务开始时使用。*

- getLibraryStats(libraryName-目标知识库的名称: string, reason-调用此工具的简要原因: string?)
  统计知识库的整体概况，包括实体总数、标签分布和关系类型分布。
  返回 XML 报告，包含 `statistics` 节点，下列出 total_entities, top_tags, top_relation_types 等统计数据。

- loadManual(libraryName-目标知识库的名称: string, reason-调用此工具的简要原因: string?)
  读取知识库的 `meta.md` 文件，获取全局描述、规则和指导原则。
  返回 XML 报告，包含 `manual_content` 节点，内含完整的 Markdown 文本。
  开发 ✅

- inspectEntityTypes(libraryName-目标知识库的名称: string, reason-调用此工具的简要原因: string?)
  列出知识库中所有已知的实体类型（通常基于 Frontmatter 或文件路径推断）。
  返回 XML 报告，列出每个类型及其对应的实体数量。

- inspectMetadataKeys(libraryName-目标知识库的名称: string, reason-调用此工具的简要原因: string?)
  扫描所有实体，收集曾被使用过的元数据键名（Frontmatter Keys）。
  返回 XML 报告，列出所有发现的键名列表，用于辅助 `findEntitiesByMetadata` 调用。

## 2. 实体检索 (Entity Retrieval)
*用于根据名称、元数据或内容特征查找特定的实体。*

- searchEntitiesByName(libraryName-目标知识库的名称: string, entityNameGlobs-遵循 FileGlobPattern 的文件名模式列表: FileGlobPattern[], reason-调用此工具的简要原因: string?)
  根据文件名（实体名）模式模糊查找实体。
  返回 XML 报告，包含匹配的实体名称列表。

- findEntitiesByMetadata(libraryName-目标知识库的名称: string, metadataQuery-遵循 MetadataQuery 的元数据筛选对象: MetadataQuery, reason-调用此工具的简要原因: string?)
  根据 Frontmatter 中的键值对查找实体。支持精确匹配。
  返回 XML 报告，包含匹配的实体名称列表。

- searchEntitiesByContent(libraryName-目标知识库的名称: string, contentGlob-遵循 TextGlobPattern 的内容匹配模式: TextGlobPattern, reason-调用此工具的简要原因: string?)
  在实体正文中搜索匹配特定模式的内容行。
  返回 XML 报告，列出匹配的实体名称、所在的章节标题以及匹配的行内容。

- searchEntitiesGlobally(libraryName-目标知识库的名称: string, pattern-遵循 TextGlobPattern 的全局匹配模式: TextGlobPattern, reason-调用此工具的简要原因: string?)
  (尚未实现) 在文件名、元数据和正文中进行全局模式匹配。
  返回 XML 报告，包含匹配的实体名称及其匹配上下文。

## 3. 实体阅读 (Entity Reading)
*在找到实体后，用于获取其详细内容（全文、目录或特定章节）。*

- loadEntities(libraryName-目标知识库的名称: string, entityNames-要加载的实体名称列表: string[], reason-调用此工具的简要原因: string?)
  读取一个或多个实体的完整内容（Frontmatter + 正文）。
  返回 XML 报告，每个实体作为一个节点，包含其完整 Markdown 内容。

- loadEntitiesToc(libraryName-目标知识库的名称: string, entityNames-要获取目录的实体名称列表: string[], reason-调用此工具的简要原因: string?)
  解析并返回一个或多个实体的目录结构（TOC），仅包含标题层级。
  返回 XML 报告，以嵌套结构展示每个实体的标题树。

- loadEntitiesSections(libraryName-目标知识库的名称: string, entityNames-目标实体名称列表: string[], sectionQueries-遵循 SubstringQuery 的标题过滤列表: SubstringQuery[], reason-调用此工具的简要原因: string?)
  读取一个或多个实体中，标题包含指定子字符串的章节内容（不区分大小写）。
  返回 XML 报告，包含匹配章节的完整文本（标题 + 正文）。

## 4. 实体管理 (Entity Management)
*涉及实体文件本身的生命周期操作：创建、重命名、合并和删除。*

- createEntities(libraryName-目标知识库的名称: string, entities-包含名称和内容的新实体列表: {name: string, content: string}[], reason-创建原因: string?)
  创建新的实体文件。如果文件已存在，操作将失败。
  返回 XML 报告，列出成功创建的实体和失败原因。

- renameEntity(libraryName-目标知识库的名称: string, oldName-当前实体名称: string, newName-新的实体名称: string, reason-重命名原因: string?)
  重命名实体文件，并自动更新所有指向该实体的引用（WikiLinks）。
  返回 XML 报告，指示操作结果及受影响的链接数量。

- mergeEntities(libraryName-目标知识库的名称: string, sourceNames-要被合并并删除的源实体名称列表: string[], targetName-接收内容的现有目标实体名称: string, reason-合并原因: string?)
  将多个源实体的内容追加到目标实体末尾，并删除源实体。自动处理链接重定向。
  返回 XML 报告，包含合并后目标实体的完整内容概要。

- trashEntities(libraryName-目标知识库的名称: string, entityNames-要移入回收站的实体名称列表: string[], reason-删除原因: string?)
  将实体文件移动到回收站目录，而非永久删除。
  返回 XML 报告，列出成功移动的实体列表。

## 5. 内容编辑 (Content Editing)
*用于修改实体内部的文本内容（章节级别）。*

- addContentToSection(libraryName-目标知识库的名称: string, entityName-目标实体名称: string, inSection-目标章节的 SectionHeading (不含#): SectionHeading, newContent-要追加的 Markdown 文本: string, reason-修改原因: string?)
  将内容追加到指定章节的末尾。必须提供精确的章节标题。
  返回 XML 报告，指示操作成功或失败。

- replaceSection(libraryName-目标知识库的名称: string, entityName-目标实体名称: string, oldHeading-现有章节的 SectionHeading: SectionHeading, newHeading-新章节的 SectionHeading: SectionHeading, newBodyContent-新的章节正文内容: string, reason-修改原因: string?)
  完全替换一个章节（包括标题和正文）。若 `oldHeading` 未找到，操作将失败。若要删除章节，可不提供 `newBodyContent` 或将其留空。
  返回 XML 报告，指示替换操作的结果。

## 6. 关系管理 (Relation Management)
*用于处理知识图谱中的边（Edge），即实体之间的连接。*

- createRelations(libraryName-目标知识库的名称: string, relations-要创建的关系对象列表: RelationObject[], reason-创建原因: string?)
  在实体间建立明确的语义关系。
  返回 XML 报告，列出成功创建的关系。

- deleteRelations(libraryName-目标知识库的名称: string, relations-要删除的关系对象列表: RelationObject[], reason-删除原因: string?)
  删除实体间已存在的语义关系。
  返回 XML 报告，列出成功删除的关系。

- getRelatedEntities(libraryName-目标知识库的名称: string, entityName-中心实体名称: string, direction-关系方向(incoming/outgoing/both): string?, relationType-过滤特定关系类型: string?, reason-查询原因: string?)
  查询与指定实体相连的其他实体。
  返回 XML 报告，列出相关实体名称及其关系类型。

- findRelations(libraryName-目标知识库的名称: string, toEntity-目标实体名称: string?, relationType-关系类型: string?, reason-查询原因: string?)
  (尚未完全实现) 搜索符合条件的关系（不限起点）。
  返回 XML 报告，列出匹配的关系列表。

- garbageCollectRelations(libraryName-目标知识库的名称: string, dryRun-是否仅报告而不执行删除: boolean?, reason-清理原因: string?)
  (尚未实现) 扫描并清理指向不存在实体的断裂关系。
  返回 XML 报告，列出被清理（或将被清理）的关系。

## 7. 系统维护 (System Maintenance)
*涉及知识库的配置修改和数据安全备份。*

- backupLibrary(libraryName-要备份的知识库名称: string)
  创建当前知识库的完整备份（zip 或 tar 归档）。
  返回 XML 报告，包含备份文件的绝对路径和大小。

- addManualSection(libraryName-目标知识库的名称: string, sectionTitle-新章节的标题: string, newContent-新章节的内容: string, reason-修改原因: string?)
  在 `meta.md` 文件末尾追加新的章节。
  返回 XML 报告，指示操作成功。

- replaceManualSection(libraryName-目标知识库的名称: string, sectionTitle-目标章节的 SectionHeading: SectionHeading, newContent-新的 Markdown 内容: string, reason-修改原因: string?)
  更新 `meta.md` 文件中的特定章节，用于修改全局规则或描述。必须提供精确的章节标题。如果 `newContent` 为空字符串，则删除该章节。
  返回 XML 报告，包含更新后的内容。
  开发 ✅


# 3. 工具描述术语表 (Glossary)

本节定义了工具名称、参数和描述中使用的核心术语，以确保 LLM 交互的语义一致性。

## 名词 (Nouns)
- FileGlobPattern  `string`
  用于匹配文件名的模式。支持 `*` (任意序列) 和 `?` (单字符)。不支持递归 `**`。
- TextGlobPattern  `string`
  用于匹配正文内容的模式。支持 `*` 和 `?`。不支持高级正则语法。
- SubstringQuery  `string`
  用于匹配章节标题的子字符串。不区分大小写，包含即匹配。
- MetadataQuery  `object`
  用于查询 Frontmatter 元数据的键值对对象（Key-Value）。
- RelationObject  `object`
  描述实体间关系的结构化对象，包含 `source`, `type` (谓词), `target`。
- SectionHeading  `string`
  章节标题文本，**不包含** Markdown 的 `#` 符号。必须**精确匹配**。
- Front Matter  `string[]`
  位于 Markdown 文件顶部的元数据块，由 `---` 分隔。

## 动词 (Verbs)
- Create (创建)  `action`
  从无到有生成全新的实体、关系或文件。
- Retrieve (检索)  `action`
  根据条件查找并返回名称列表，是加载的前置步骤。
- Load (加载)  `action`
  根据名称读取并返回具体的文本内容（全文、章节或目录）。
- Edit (编辑)  `action`
  修改实体内部文本，不改变其实体身份。
- Connect (连接)  `action`
  建立或断开实体之间的语义关系（图谱边）。
- Organize (组织)  `action`
  通过重命名、合并、删除等操作重构知识库结构。
- Backup (备份)  `action`
  创建知识库快照。
- Find  `method`
  根据**确定性、结构化**条件查找资源（如元数据、关系）。
- Search  `method`
  根据**模糊、非结构化**模式进行全文搜索（如 Glob, Regex）。
- Replace  `method`
  原地替换现有内容。
- Trash  `method`
  将文件软删除至回收站。
- Merge  `method`
  将多个内容合并为一个整体。

## 概念 (Concepts)
- Library (知识库)  `unit`
  工作基本单元，包含实体集合与唯一的 `meta.md`。
- Meta / Manual (手册)  `file`
  指 `meta.md`，记录知识库的全局规则、用途和指导原则。
- XML Report  `output`
  工具返回的结构化 XML 结果。
- Reason  `parameter`
  调用工具的简要目的说明，用于审计。
