# 项目术语表 (Glossary)

本文档旨在为 `mcp-server-memories-off` 项目建立一套“通用语言”(Ubiquitous Language)，对项目中的核心名词和动词进行唯一且精确的定义。所有新的开发和重构都应遵循此处的约定。

---

## 1. 名词 (Nouns)

本节定义了项目中的核心数据实体和概念。

### 1.1. 文本与内容结构

| 术语 | 类型 | 描述 |
| :--- | :--- | :--- |
| **`Line`** | `string` | 文件中的**单行文本**。是内容的最基本单位。 |
| **`Content`** | `string[]` | 由多行 `Line` 组成的**文本块**。用于表示文件或章节的正文。 |
| **`Heading`** | `{ level: number, text: string }` | 代表一个 Markdown **标题**。在代码中通常表现为 `TocItem` 类型。 |
| **`Section`** | `{ heading: Heading, content: Content }` | 一个完整的**逻辑章节**，由一个 `Heading` 及其紧随其后的 `Content` 组成。 |
| **`TOC`** | `Heading[]` | **Table of Contents** 的缩写，指整个文件的**标题列表**。在代码中通常表现为 `TocList` 类型。 |
| **`Front Matter`**| `string[]` | 位于 Markdown 文件顶部的元数据块，由 `---` 分隔。 |

### 概念

Library 知识库 - 工作基本单元，包含一系列文件，仅有一个 Meta。

Thing 文件 - 包括 Entity, Journal, Meta 三种。

Entity 实体 - 知识库中的一个具体事物，对应一个 Markdown 文件。

Journal 日志 - 知识库中的日志，对应一个 markdown 文件。

Meta 元数据 (Manual) - 知识库的元数据文件 (`meta.md`)，也作为高级别的**手册**使用，记录知识库的用途、规则等。

### 1.2. 路径与标识符

| 术语 | 类型 | 描述 |
| :--- | :--- | :--- |
| **`LibraryName`** | `string` | **知识库名称**，例如 "MyProjectDocs"。 |
| **`ThingName`** | `string` | **事物名称**，通常指不带文件后缀名的实体或记录名称。 |
| **`FileRelativePath`**| `string` | 相对于知识库根目录的**文件相对路径**，例如 "entities/MyEntity.md"。 |
| **`FileAbsolutePath`**| `string` | 文件在文件系统中的**绝对路径**。 |
| **`FolderAbsolutePath`**| `string` | 文件夹在文件系统中的**绝对路径**。 |

## 2. 动词 (Verbs)

本节定义了项目中用于函数和方法命名的操作动词，以确保其含义的一致性。

| 术语              | 描述                                  | 示例                                               |
|:----------------|:------------------------------------|:-------------------------------------------------|
| **`get`**       | 通过**唯一标识符**精确获取**单个**资源。            | `getEntityById`, `getSettings`                   |
| **`list`**      | **枚举**一类资源，通常返回一个列表。                | `listEntities`, `listAllEntityTypes`             |
| **`find`**      | 根据一组**结构化、确定性**的条件查找资源。常用于基于元数据的查询。 | `findEntitiesByMetadata`, `findRelations`        |
| **`search`**    | 根据**模糊、非结构化**的模式（如全文搜索、正则表达式）搜索资源。  | `searchInContents`, `searchEntityContentByRegex` |
| **`add`**       | 向某个集合或文件中**追加**新内容。                 | `addContentToSection`                            |
| **`create`**    | **创建**一个全新的、独立的对象或文件。               | `createFile`                                     |
| **`delete`**    | 从某个集合或文件中**移除**现有内容。                | `deleteContentFromSection`                       |
| **`move`**      | **移动**或**重命名**一个文件或资源。              | `renameFile`                                     |
| **`trash`**     | 将文件或资源**软删除**至回收站。                  | `trashFile`, `trashEntity`                       |
| **`write`**     | 将内容**覆写**到文件中。                      | `writeFileContent`                               |
| **`read`**      | 从文件中**读取**内容。                       | `readFileContent`                                |
| **`normalize`** | 将字符串**标准化**为一种统一的、可比较的格式。           | `normalize(heading)`                             |
| **`check`**     | 检查某个条件是否为真，通常返回布尔值。                 | `checkFileExists`                                |
| **`match`**     | 使用 glob **匹配** 现有对象。                | `matchToc`                                       |
| **`replace`**   | 原地 **替换** 现有内容。                     | `replaceSection`                                 |
| **`insert`**    | 在指定位置 **插入** 新内容。                   | `insertLineAt`                                   |
| **`locate`**    | **定位** 某个资源的位置，通常返回路径或索引。           | `locateEntityFile`                               |
| **`merge`**     | 将多个内容 **合并** 为一个整体，通常移除重复           | `mergeContents`                                  |
| **`garbageCollect`** | **清理**悬空、无效的资源。 | `garbageCollectRelations` |

## 3. 工具描述用词

本节定义了在工具描述中使用的核心动词，旨在为 LLM 提供一套无歧义、语义清晰的操作指令。

| 术语 | 英文 | 定义 (面向 LLM) |
| :--- | :--- | :--- |
| **创建** | Create | 从无到有地生成一个全新的、独立的对象，例如实体或关系。 |
| **检索** | Retrieve | 根据一组指定的条件（如元数据、内容模式、关系）来查找并返回一个或多个**实体标识符（名称）**的列表。这是“加载”的前置步骤。 |
| **加载** | Load | 当实体标识符（名称）已知时，读取并返回该实体的**具体内容**（如全文、章节、目录）。 |
| **编辑** | Edit | 修改一个已知实体的**内部文本内容**，而不改变其实体身份或与其他实体的结构关系。 |
| **连接** | Connect | 创建或删除实体之间的**关系**，以建立或断开知识图谱中的连接。 |
| **组织** | Organize | 通过重命名、合并、删除等操作来**重构**知识库的整体结构或实体身份。 |
| **备份** | Backup | 创建整个知识库的快照，用于恢复。 |
