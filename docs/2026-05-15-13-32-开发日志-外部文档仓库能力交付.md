# 开发日志: 外部文档仓库 (External Document Repository) 能力交付

## 1. 概述 (Overview)
本次开发为 `memories-off` 引入了**外部文档仓库 (External Document Repository)** 能力。该功能允许用户在不改变现有文档结构的情况下，通过 `memocli` 快速检索和阅读外部知识库（如 Obsidian）中的文档。

## 2. 实现的功能点 (Features)
1.  **配置解析**: 扩展了 `LibraryContext`，支持从 `meta.md` 中读取 `external document repo`、`external document extensions` 和 `external document exclude dirs` 配置。
2.  **模糊检索 (`find-doc-by-name`)**:
    - 支持按文件名模糊匹配。
    - 自动应用扩展名过滤和子目录排除规则。
    - 返回文档的相对路径及长度信息。
3.  **精确读取 (`read-doc-by-name`)**:
    - 支持通过相对路径直接读取。
    - 支持仅凭唯一文件名进行智能定位。
    - 严守只读原则，不修改外部文件。

## 3. 主要代码变更 (Key Changes)
- `memories-off-skill/scripts/utility/schema_define.py`:
    - 新增 `ExternalRepoConfig` 数据类。

- `memories-off-skill/scripts/commands/find_doc_by_name.py`: 新增子命令脚本。
- `memories-off-skill/scripts/commands/read_doc_by_name.py`: 新增子命令脚本。

## 5. 后续建议 (Suggestions)
- 当前检索仅限于文件名。如果后续有需求，可以考虑基于 `grep` 或 `ripgrep` 增加轻量级的内容搜索功能，但需保持“被动触发”原则。
- 可以考虑在 `explore` 命令中增加对外部仓库状态的简要统计。
