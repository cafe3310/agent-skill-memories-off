# 开发日志: 外部文档仓库 (External Document Repository) 功能全线贯通

## 1. 背景 (Background)
用户希望在 `memories-off` 知识库的基础上增加一个「只读文档库」的能力，以便快速检索和阅读外部文档（如 Obsidian 仓库），而无需将它们转化为标准的 `memories-off` 实体。该功能强调只读、被动检索以及风格的一致性。

## 2. 任务执行记录 (Action Log)
1. **需求定义**: 明确了 **外部文档仓库 (External Document Repository)** 的概念，定义了 `meta.md` 中的配置项。
2. **核心逻辑实现**:
    - 扩展 `LibraryContext`，支持解析 `external document repo`、`external document extensions` 和 `external document exclude dirs`。
    - 优化了 Key 的命名风格，采用空格分隔以匹配项目既有规范。
3. **工具集开发**:
    - `find-doc-by-name`: 实现模糊匹配文件名、路径解析及文件大小统计。移除了冗余的 `--names-only` 参数，并优化了 metadata 获取性能。
    - `read-doc-by-name`: 实现智能定位文档内容。支持优先文件名匹配，仅在冲突时要求路径。
4. **系统集成**: 运行 `install.py` 重新注册子命令，更新了帮助系统。
5. **验证**: 
    - 编写并执行了集成测试集。
    - 成功为用户真实的 `cafe3310-db/memories-off` 库配置并测试了 `obsidian-agi` 仓库的接入。

## 3. 实现细节与优化 (Implementation Details)
- **性能**: `find-doc-by-name` 不再全量读取文件内容来统计长度，而是通过 `os.path.getsize` 获取 Bytes 大小，显著提升了扫描效率。
- **一致性**: 配置项从 `ext_doc_repo_root` 风格更名为 `external document repo`，确保元数据 Key 的视觉统一。
- **易用性**: `read-doc-by-name` 的例子和描述被微调为“文件名优先”，降低了用户（及 Agent）的记忆负担。

## 4. 交付产出 (Deliverables)
- **代码**: `schema_define.py`, `find_doc_by_name.py`, `read_doc_by_name.py`。
- **文档**: 更新了 `README.md`、`TODO.md` 以及对应的需求文档。
- **测试**: `tests/2026-05-15-13-40-testsuite/`。
