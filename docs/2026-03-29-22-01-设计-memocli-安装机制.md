# 需求与设计: memocli 安装与功能转发机制

## 1. 背景
用户 cafe3310 在 `memories-off` 项目中提出需要一个「安装」功能，旨在通过在系统 `PATH` 中创建一个名为 `memocli` 的快捷命令，实现对项目脚本（位于 `memories-off-skill/scripts/`）的便捷调用。

## 2. 目标
- 提供一个 `install.sh` 脚本，自动化完成 `memocli` 的配置与安装。
- `memocli` 应支持 `memocli <action> <args>` 格式，将请求转发给对应的 Python 脚本。
- 实现智能路径识别：在知识库根目录下执行时，若未指定 `--path`，自动注入 `--path .`。

## 3. 详细设计

### 3.1 转发逻辑 (Action Dispatcher)
`memocli` 脚本将执行以下逻辑：
1. **获取参数**：第一个参数为 `action`，剩余为 `args`。
2. **路径构建**：寻找 `<SCRIPTS_DIR>/<action>.py`。
3. **环境注入**：
   - 检查当前目录是否存在 `meta.md`。
   - 检查 `args` 中是否已包含 `--path` 或 `-p`。
   - 若满足「在库中」且「无路径参数」，则自动追加 `--path .`。
4. **执行**：调用 `python3 <SCRIPT_PATH> <ARGS>`。

### 3.2 安装程序 (`install.sh`)
1. **路径解析**：获取当前 `scripts` 目录的绝对路径。
2. **目标定位**：
   - 优先检查 `/usr/local/bin`。
   - 若无权限，使用 `~/.local/bin`。
3. **脚本生成**：使用 `cat <<EOF` 动态生成包装器，并注入绝对路径。
4. **权限处理**：`chmod +x`。
