# 任务：标准化文件命名规范

- **任务来源**: 用户于 2026-01-04 提出的需求。
- **关联文档**: `docs/2026-01-04-21-40-knowledge-base-file-structure.md`
- **优先级**: 高

---

## 1. 任务概述

为了确保知识库文件的跨平台兼容性、可读性和一致性，需要对项目中所有自动生成的文件名（实体、回收站文件、备份文件）实施一套标准化的命名和格式化规则。

当前，这些规则仅在文档中定义，需要通过代码修改来强制执行。

## 2. 开发项 (Development Items)

### 2.1. 实现实体名称的自动格式化

- **目标**: 确保所有实体文件的名称都符合预定义的字符和格式规范。
- **开发内容**:
    1.  在 `src/basics/text.ts` 中创建一个新的工具函数 `sanitizeEntityName(name: string): string`。
    2.  此函数需要实现以下逻辑：
        - 仅允许 CJK 字符、英文字母、数字和中横线 (`-`)。
        - 所有不符合上述规则的字符（如空格、标点符号）将被替换为单个中横线。
        - 连续的多个中横线将被合并为一个。
        - 文件名首尾的中横线将被移除。
    3.  在 `src/runtime.ts` 中，修改 `getEntityFilePath` 函数，使其在拼接最终文件名之前，调用 `sanitizeEntityName` 来处理 `entityName`。
    4.  在 `src/basics/text.test.ts` 中为 `sanitizeEntityName` 函数添加单元测试，覆盖各种边界情况。

### 2.2. 调整回收站文件的命名格式

- **目标**: 使回收站中的文件可以按删除时间自然排序。
- **开发内容**:
    1.  在 `src/runtime.ts` 中，修改 `generateEntityTrashPath` 函数。
    2.  将文件名格式从 `[原文件名]_[时间戳].md` 调整为 `[时间戳]_[格式化后的原文件名].md`。
    3.  确保在拼接前，原文件名也经过 `sanitizeEntityName` 函数的处理。

### 2.3. 调整备份文件的命名格式

- **目标**: 使备份文件可以按备份时间自然排序。
- **开发内容**:
    1.  在 `src/runtime.ts` 中，修改 `generateBackupPath` 函数。
    2.  将文件名格式从 `[知识库名称]-backup-[时间戳].zip` 调整为 `backup_[时间戳]_[知识库名称].zip`。
