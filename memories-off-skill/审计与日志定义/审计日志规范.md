# 审计与日志定义: Git Audit Specification

本文档定义了 `memory-skill` 中利用 Git 进行变更审计的规范。

---

## 1. 核心机制 (Core Mechanism)

- **版本控制**: 知识库根目录必须初始化为 Git 仓库。
- **强制提交**: 所有的 `Mutation` 操作（增删改）完成后，必须通过脚本或命令执行 `git commit`。
- **审计查询**: 通过 `git log` 或 Git 客户端查看所有历史记录。

## 2. 提交信息规范 (Commit Message Standard)

为了确保审计信息的完整性，提交信息应采用以下格式：

`[Action] [Target]: [Reason]`

### 字段说明：
- **Action**: 执行的操作名称（如 `edit`, `create`, `rename`）。
- **Target**: 受影响的实体文件名。
- **Reason**: 执行操作的原因（强制提供）。

### 示例：
- `edit 人物-cafe3310: 更新了近期参与的项目列表`
- `create 宠物-咪咪: 新增猫咪基础档案`
- `rename 物品-渴望猫粮 -> 物品-Orijen: 统一使用英文原名`
