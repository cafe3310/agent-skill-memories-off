# 关联定义: Relation Types

本文档定义了 `memories-off` 中实体间显式语义关系的表示方式、命名规范及处理规则。

---

## 1. 显式关系表示 (Explicit Relations)

显式关系定义在实体的 Front Matter 中，具有强语义特征，可供 Agent 直接进行结构化检索和逻辑推理。

### 1.1 `relation as [predicate]`
- **职能**: 定义“当前实体”对“目标实体”的特定语义指向。
- **格式**: `relation as [谓词]: [目标实体名称]`。
- **多值支持**: 多个目标实体以英文逗号 (`,`) 分隔。
- **谓词规范化**: 系统会自动执行标准化（小写化，空格/特殊字符转为下划线）。例如 `relation as Member Of` 会被解析为 `relation as member_of`。

## 2. 关系方向 (Relation Direction)

- **Outgoing (出站)**: 从当前实体指向外部的关系。
- **Incoming (入站)**: 从外部指向当前实体的关系（由系统通过全局扫描汇总）。
- **分析工具**: 使用 `memocli get-relations --entity <name>` 可一键获取指定实体的完整出/入站关系网。

## 3. 谓词建议 (Predicate Suggestions)

- **层级**: `is_a`, `part_of`, `child_of`。
- **归属**: `owned_by`, `member_of`。
- **协作**: `reported_to`, `assigned_to`, `managed_by`。
- **通用**: `related_to` (关联), `dependent_on` (依赖)。

## 4. 自动维护与治理

- **目标标准化**: 关系中的目标实体名会被自动转为标准 Slug 形式。
- **重命名同步**: 执行 `memocli rename` 时，全库所有实体的关系目标将同步更新。
- **健康检查**: 使用 `memocli doctor --rule-audit-links` 扫描并清理指向不存在实体的陈旧关系条目。

## 5. 检索与检索

- **精确过滤**: 使用 `memocli search --rel <谓词正则>:<目标正则>` 可以基于关系进行高效筛选。
