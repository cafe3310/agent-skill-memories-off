# 关联定义: Link Logic (WikiLinks)

本文档定义了 `memories-off` 中基于 WikiLinks 的隐式链接逻辑及维护机制。

---

## 1. WikiLinks 定义

WikiLinks 是指在实体正文或元数据中使用 `[[目标实体名称]]` 语法引用的关联。

### 1.1 语法规则
- **标准格式**: `[[TargetName]]`。系统将其解析为指向 `entities/TargetName.md` 的链接。
- **显示名支持**: `[[TargetName|DisplayName]]`。
- **标准化**: 系统会将 `TargetName` 自动转为标准 Slug 形式（字母、数字、中文、下划线及单中横线）。

## 2. 链接发现与分析

- **库级探索**: `memocli explore` 会自动提取 `meta.md` 中的所有 WikiLinks，并预览其目标实体的元数据。
- **全方位检索**: `memocli search` 支持检索正文中的 WikiLinks。
- **双向关系**: `memocli get-relations` 不仅分析显式关系，未来也将支持汇总 WikiLinks 带来的隐式引用。

## 3. 核心维护逻辑: 重命名同步

这是 `memories-off` 保持图谱完整性的核心能力，由 `memocli rename` 驱动。

### 3.1 自动扫描与替换
- **全库覆盖**: 重命名操作会扫描 `entities/` 目录下的所有文件及根目录下的 `meta.md`。
- **精确替换**: 自动将所有 `[[OldName]]` 或 `[[OldName|...]]` 准确更新为指向新名称的链接。

## 4. 健康治理: Doctor

通过 `memocli doctor` 可以批量治理全库的链接健康度：
- **`--rule-normalize-name`**: 自动将正文中非标形式的 WikiLinks 纠正为标准 Slug 形式。
- **`--rule-audit-links`**: 发现并向用户报告指向不存在文件的“死链”。

## 5. 与显式关系的区别

| 特性 | WikiLinks (隐式) | `relation as` (显式) |
| :--- | :--- | :--- |
| **存储位置** | Body (正文) | Front Matter (元数据) |
| **语义强度** | 弱 (仅表示存在提及) | 强 (定义了具体的谓词属性) |
| **适用场景** | 叙述性文本、随手记录 | 结构化图谱、业务依赖、管理关系 |
| **治理工具** | `doctor --rule-normalize-name` | `doctor --rule-audit-links` |
