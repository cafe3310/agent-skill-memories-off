# 知识库结构定义: Meta Specification (meta.md)

本文档定义了知识库核心手册 `meta.md` 的内部结构规范。`meta.md` 是 Agent 理解知识库上下文、规则、Schema 和架构骨架的首要入口。

---

## 1. 核心目标 (Goals)

- **上下文对齐**: 提供知识库用途的全局描述。
- **Schema 定义**: 集中定义该库的实体类型、元数据字段。
- **架构骨架 (Skeleton)**: 通过 WikiLinks 链接最关键的基石实体，形成导航地图。
- **规则约束**: 规定 Agent 在此库中进行编辑、命名或关联时应遵循的指导原则。

## 2. 标准章节结构 (Standard Sections)

### 2.1 `# 全局描述 (Global Description)`
- 简述知识库主题。
- **架构链接**: 在此处或专门的章节中使用 `[[WikiLinks]]` 链接起整个知识库架构的“基石实体”。`memocli explore` 会自动解析这些链接并为 Agent 提供它们的元数据预览。

### 2.2 `# 实体类型定义 (Entity Types)`
- 详细列出该库中使用的 `entity type` 及其适用场景。

### 2.3 `# 元数据规范 (Metadata Schema)`
- 说明自定义字段的含义。

### 2.4 `# 指导原则与约束 (Guidelines & Constraints)`
- 定义特定的命名、关联及内容记录偏好。

## 3. 维护规范 (Maintenance Rules)

- **获取**: 运行 `memocli explore` 以结构化方式获取手册内容。
- **编辑**: Agent 可以直接使用 `write_file` 对其进行手术式修改，但完成后必须立即运行 `memocli commit --reason "更新手册规范"`。
- **动态演进**: 随着知识库的发展，Agent 应根据发现的新模式主动建议更新 `meta.md`。

## 4. 缺失处理 (Missing File Handling)

- 若知识库中不存在 `meta.md`，Agent 应调用 `memocli init` 进行初始化。
