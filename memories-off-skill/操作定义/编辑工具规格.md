# 操作定义: Editing Tools

本文档定义了用于修改实体内部内容的原子化工具规格。

---

## 1. 实体内容编辑 (Content Editing)

### 1.1 `append-update` (首选编辑方式)
- **职能**: 向实体文件末尾追加结构化的缓冲更新块，由后续整理脚本消化。支持一并更新关系。
- **[▶️执行方式]**:
  - 命令行模式: `memocli append-update --entity <name> --content <text> --reason <reason>`
  - STDIN 模式 (推荐): 
    ```bash
    echo "## 2026-04-20 讨论记录
    - 参会人: [[cafe3310]], [[Gemini]]
    - 关键结论: 引入了 STDIN 传输模式以避开转义坑。
    - 风险点: 注意交互式环境下不要误入等待状态。" | memocli append-update --entity "项目进展" --content-stdin --reason "同步会议纪要"
    ```
- **逻辑说明**: 
  - 追加的内容会被自动包裹在带有双时间戳的 `UPDATE_BLOCK` 中。
  - 会自动触发 WikiLink 标准化和标题层级（H2）标准化。
  - `--content` 与 `--content-stdin` 互斥。

### 1.2 `consolidate-updates` (梦境整理)
- **职能**: [高影响] 解析实体末尾的所有更新块，并按逻辑合并到正式章节中。
- **[▶️执行方式]**: `memocli consolidate-updates --path . --reason <reason>` **(已内置 commit)**
- **触发时机**: 当一个实体累积了多个更新块，或用户显式要求“整理图谱”时执行。Agent 不得在未获授权时擅自运行。

## 2. 元数据维护 (Metadata Maintenance)

### 2.1 `manage-relations` (关系微调)
- **职能**: 精确管理实体的显式语义关系。
- **[▶️执行方式]**: `memocli manage-relations --source <name> [--add-rel-out ...] [--add-rel-in ...] [--remove-rel-out ...] --reason <reason>` **(已内置 commit)**

### 2.2 `commit` (手工审计提交)
- **职能**: 当 Agent 通过 `read_file` / `write_file` 或手动编辑 `meta.md` 后，必须使用此工具补齐审计日志。
- **[▶️执行方式]**: `memocli commit --path . --reason <reason>`

---

## 3. 操作准则 (Guidelines)

- **非破坏性编辑**: 始终通过 `append-update` 而非全量重写来修改文件。这确保了变更的可回滚性和结构完整性。
- **强制审计**: 必须为每一次变更（哪怕只是微调关系）提供清晰且具备上下文的 `--reason`。
- **标题意识**: 由于知识库实施了 H2 标准化，Agent 在提交 `--content` 时无需纠结层级，系统会自动将其中的标题拍平为 `##`。
- **自动转码**: 所有通过 CLI 写入的 `[[WikiLinks]]` 都会经历自动转码（如空格转连字符），Agent 应优先相信系统的转码结果。
