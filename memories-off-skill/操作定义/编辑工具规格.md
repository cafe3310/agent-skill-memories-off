# 操作定义: Editing Tools

本文档定义了用于修改实体内部内容的原子化工具规格。

---

## 1. 实体内容编辑 (Content Editing)

### 1.1 `appendUpdateBlock`
- **职能**: 向实体文件末尾追加缓冲更新块，由后续整理脚本消化。
- **[▶️执行方式]**: `python scripts/append_update.py --path <path> --entity <entity> [--target_heading <heading>] --action <action> --content <content> --reason <reason>` **(已内置 commit)**
- **推荐策略**: 这是 Agent 修改已有实体的**首选方式**，可绝对避免内容丢失。

### 1.2 `consolidateUpdates` (系统触发)
- **职能**: 解析全库的更新块并合并到正文。
- **[▶️执行方式]**: `python scripts/consolidate_updates.py --path <path> [--reason <reason>]` **(已内置 commit)**
- **触发时机**: 用户显式命令“梦境整理”时执行。

### 1.3 `addContentToSection`
- **职能**: 将新的 Markdown 文本追加到指定章节的末尾。
- **[▶️执行方式]**: `python scripts/append_update.py --path <path> --entity <entity> --target_heading <section> --action append --content <content> --reason <reason>` **(已内置 commit)**
- **逻辑说明**: 该操作现在通过“缓冲追加”实现。内容将暂时存放在文件末尾的更新块中，待“梦境整理”时自动合并至目标章节末尾。

### 1.4 `replaceSection`
- **职能**: 完全替换现有章节（包括标题和正文）。
- **[▶️执行方式]**: `python scripts/append_update.py --path <path> --entity <entity> --target_heading <oldHeading> --action replace --content <newContent> --reason <reason>` **(已内置 commit)**
- **逻辑说明**: 该操作现在通过“缓冲替换”实现。原章节内容在“梦境整理”前保持不变，新内容暂时存放在末尾更新块中。


## 2. 系统元数据编辑 (System Manual Editing)

### 2.1 `addManualSection`
- **职能**: 在 `meta.md` 末尾追加新章节。
- **[▶️执行方式]**: Agent 直接在 `meta.md` 使用 `replace` 或追加。**(需手动调用 `scripts/commit.py`)**
- **参数**: `libraryName`, `sectionTitle`, `newContent`, `reason`。

### 2.2 `replaceManualSection`
- **职能**: 替换 `meta.md` 中的特定章节。
- **[▶️执行方式]**: Agent 直接在 `meta.md` 使用 `replace`。**(需手动调用 `scripts/commit.py`)**
- **参数**: `libraryName`, `sectionTitle`, `newContent`, `reason`。

---

## 3. 操作准则 (Guidelines)

- **非破坏性编辑**: 始终通过 `replaceSection` 而非全量重写来修改大文件，以防丢失未预期的内容。
- **强制审计**: 必须为每一次编辑提供清晰的 `reason`。
- **精确标题匹配**: `inSection` 和 `oldHeading` 必须与文件中存在的标题完全一致（不含 `#`），否则操作失败。
- **原子性**: 每次工具调用应仅完成一个逻辑上的修改动作。
- **XML 反馈**: 报告必须明确指示修改前后的状态差异。
