# 操作定义: Retrieval & Loading Tools

本文档定义了用于查找和读取知识库内具体实体的工具规格。

---

## 1. 实体检索 (Retrieval Tools)

### 1.1 `search` (统一检索器)
- **职能**: 根据模式（正则表达式）在全库范围内检索实体。支持精准的类型、关系过滤及别名感知。
- **[▶️执行方式]**: `memocli search "<pattern>" [options]`。(无需 commit)
- **输入参数**:
  - `pattern` (可选): 搜索文本。支持正则表达式。
  - `-n, --name`: 仅匹配实体文件名及 Frontmatter 中的 `aliases` (别名)。
  - `-c, --content`: 仅匹配实体正文内容。
  - `-t, --type <regex>`: 精准过滤特定的实体类型。
  - `--rel <pred_regex:target_regex>`: 过滤声明了特定关系的实体（例如 `friend:张三`）。
  - `--names-only`: 仅输出匹配的实体名称列表，极大减少 Token 消耗。
  - `-C, --context <num>`: 匹配正文时输出上下文行数。

### 1.2 `get-relations` (关联分析)
- **职能**: 分析特定实体的双向关联网络。
- **[▶️执行方式]**: `memocli get-relations --entity <name>`。(无需 commit)
- **输出**: 列出该实体声明的所有出站关系，以及全库中所有指向该实体的入站关系。

## 2. 实体加载 (Loading Tools)

### 2.1 `load` (全量加载)
- **职能**: 精确读取并返回一系列实体的完整 Markdown 内容（含 YAML Frontmatter）。
- **[▶️执行方式]**: `memocli load --names <实体1,实体2>`。(无需 commit)
- **输出**: 结构化的 XML 报告，包含成功读取的全文以及失败的错误信息。

### 2.2 `loadEntitiesToc` (目录加载)
- **职能**: 快速获取实体的章节结构。
- **[▶️执行方式]**: `grep "^#" entities/<name>.md`。由于知识库实施了 H2 统一规范，这会返回实体名(H1)及所有业务章节(H2)。

### 2.3 `loadEntitiesSections` (定向章节加载)
- **职能**: 根据标题关键词，读取实体中特定章节的内容。
- **[▶️执行方式]**: 目前仍建议 Agent 使用 `read_file` 配合行号定位或全文读取后自行提取。未来将支持 `load --section` 指令。

---

## 3. 操作准则 (Guidelines)

- **由面到点**: 建议先使用 `search --names-only` 确定目标实体，再读取具体内容。
- **别名感知**: 即使忘记了实体的确切 Slug，也可以通过 `search --name <alias>` 利用别名系统找回主实体。
- **属性驱动**: 当需要查找“所有人物”或“所有属于某项目的实体”时，应优先使用 `search --type` 或 `search --rel` 而非全文 Grep。
