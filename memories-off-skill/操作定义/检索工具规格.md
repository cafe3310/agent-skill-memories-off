# 操作定义: Retrieval & Loading Tools

本文档定义了用于查找和读取知识库内具体实体的工具规格。

---

## 1. 实体检索 (Retrieval Tools)

### 1.1 `searchEntitiesByName`
- **职能**: 根据文件名（实体名）模式进行模糊匹配。
- **[▶️执行方式]**: `ls entities/` 后接 `grep` 或使用 `glob` 模式。(无需 commit)
- **输入**: `libraryName`, `entityNameGlobs` (模式列表，如 `*User*`)。
- **输出**: 匹配的实体名称列表 (`entities` 节点)。

### 1.2 `findEntitiesByMetadata`
- **职能**: 根据 Frontmatter 键值对查找实体（结构化查询）。
- **[▶️执行方式]**: 使用脚本扫描或 `grep` 元数据区。(无需 commit)
- **输入**: `libraryName`, `metadataQuery` (键值对对象)。
- **输出**: 匹配的实体名称列表。

### 1.3 `searchEntitiesByContent`
- **职能**: 在实体正文（Body）中搜索特定内容模式。
- **[▶️执行方式]**: `grep_search`。(无需 commit)
- **输入**: `libraryName`, `contentGlob` (内容模式)。
- **输出**: 匹配项列表，包含 `entityName`, `sectionHeading`, `matchedLine`。

### 1.4 `searchEntitiesGlobally`
- **职 neng**: 在实体名称、元数据和正文中进行全方位全局搜索。
- **[▶️执行方式]**: `grep_search`。(无需 commit)
- **输入**: `libraryName`, `pattern`。
- **输出**: 综合匹配结果报告。

## 2. 实体加载 (Loading Tools)

### 2.1 `loadEntities`
- **职能**: 读取并返回实体的完整 Markdown 内容。
- **[▶️执行方式]**: `read_file`。(无需 commit)
- **输入**: `libraryName`, `entityNames` (名称列表)。
- **输出**: 包含每个实体完整内容的报告。

### 2.2 `loadEntitiesToc`
- **职能**: 解析并返回实体的目录结构（Heading 树）。
- **[▶️执行方式]**: `grep "^#" entities/<name>.md`。(无需 commit)
- **输入**: `libraryName`, `entityNames`。
- **输出**: 嵌套结构的目录树报告 (`toc` 节点)。

### 2.3 `loadEntitiesSections`
- **职能**: 根据标题关键词，读取实体中特定章节的内容。
- **[▶️执行方式]**: `read_file` 定向行读取。(无需 commit)
- **输入**: `libraryName`, `entityNames`, `sectionQueries` (标题关键字列表)。
- **输出**: 匹配章节的完整 Markdown 块。

---

## 3. 操作准则 (Guidelines)

- **由面到点**: 建议先使用 `search/find` 工具确定目标实体，再使用 `load` 工具读取具体内容，以节省上下文 Token。
- **分阶段加载**: 对于大型实体，应优先加载 `TOC` 以了解结构，随后再通过 `loadEntitiesSections` 定向读取所需章节。
- **模糊匹配策略**: `HeadingGlob` 仅支持包含匹配（Substring），在 `search` 类操作中应遵循此规则。
