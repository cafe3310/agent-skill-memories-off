# 操作定义: Observation Tools

本文档定义了用于观察和获取知识库全景信息及进行系统健康治理的工具规格。

---

## 1. `explore` (全景探索)
- **职能**: 知识库的超级入口。在任务开始时一次性获取手册内容、架构骨架、类型分布及工具列表。
- **[▶️执行方式]**: `memocli explore --path . [--full]` (无需 commit)
- **核心模块 (XML)**:
  - `<meta-path>`: 指向 `meta.md` 的绝对路径。
  - `<meta-content>`: 手册正文全文。
  - `<core-entities>`: 手册中提及的关键实体元数据预览。
  - `<global-entity-types>`: 知识库全量实体类型及数量统计（非采样）。
  - `<global-relations>`: 知识库全量关系契约分布统计（非采样）。
  - `<cli-help>`: 完整的 `memocli` 帮助菜单。

## 2. `doctor` (系统治理与修复)
- **职能**: 诊断并修复知识库中的常见格式、命名及引用问题。
- **[▶️执行方式]**: `memocli doctor --path . --rule-xxx [--fix] --reason <reason>` **(带 --fix 时内置 commit)**
- **可用规则 (`--rule-`)**:
  - `--rule-normalize-name`: 修复实体文件名、WikiLinks 和关系目标的标准化命名。
  - `--rule-normalize-headers`: 将正文中的所有 Markdown 标题统一标准化为 H2 层级。
  - `--rule-audit-links`: 检查并清理损坏的 WikiLink 以及孤立的显式关系。
  - `--rule-fix-update-blocks`: 交互式修复不规范或老旧格式的 `UPDATE_BLOCK` 缓冲块。

## 3. `stats` (资源统计)
- **职能**: 获取知识库的全局资产报告（作为 `explore` 的轻量级补充）。
- **[▶️执行方式]**: `memocli stats --path .` (无需 commit)
- **输出**: 实体总数、类型计数、关系计数及磁盘占用摘要。

---

## 4. 操作准则 (Guidelines)

- **探索优先**: Agent 在首次进入新知识库或开始新一轮复杂任务前，**必须首先调用 `explore`**。
- **健康治理**: 周期性或在执行重命名等大规模变动后，应运行 `doctor` 配合 `--rule-xxx` 进行健康扫描，确保数据始终处于标准化状态。
- **结构化读取**: 观察类工具均返回结构化的 XML 报告，Agent 应从中提取对应的标签内容以确立认知，而非盲目 `ls` 或全量 `grep`。
