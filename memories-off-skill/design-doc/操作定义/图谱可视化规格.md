# 操作规格: 3D 知识图谱可视化 (Force-Graph)

## 1. 概述 (Overview)

本规格定义了 `memories-off` 知识库的图形化展示工具。该工具通过模拟引力场（Force-directed Layout）来呈现知识实体间的密度与关联，帮助用户直观识别知识簇、断头链接和逻辑环路。

**注意**: 该工具旨在提供宏观视野，而非执行具体的逻辑修改。

---

## 2. 核心指令 (CLI Command)

使用 `memocli graph` 指令唤起可视化界面：

```bash
# 在知识库根目录下直接运行
memocli graph

# 指定路径运行
memocli graph --path /path/to/your/library
```

### 2.1 执行流程
1. 脚本扫描整个知识库，提取实体（Nodes）及其关联（Links）。
2. 在系统临时目录生成一个自包含的 `.html` 渲染文件。
3. 自动尝试通过系统默认浏览器打开该文件。
4. 若自动打开失败，请根据终端输出的 `file://` 路径手动访问。

---

## 3. 视觉表现规范 (Visual Conventions)

### 3.1 节点 (Nodes)
- **大球体**: 代表核心实体（Entity）。
- **小球体**: 代表实体的别名（Alias）。
- **颜色**: 基于实体类型（Type）自动分配对比色。
- **标签**: 仅显示实体/别名的显示名称（Display Name），移除了类型前缀。

### 3.2 连线 (Links)
- **极短紧致线**: 表示「别名」与其「主实体」的绑定关系。
- **带箭头实线**: 表示显式定义的 `relations` 关系，箭头指向谓词目标。
- **隐式点线/细线**: 表示正文中的 `[[WikiLinks]]` 引用。

---

## 4. 物理场逻辑 (Physics Logic)

图谱内置了基于 D3.js 的物理仿真引擎：
- **类型引力 (Type-Centric Gravity)**: 相同类型的节点会自动向该类型的虚拟中心聚拢，形成明显的颜色聚类。
- **全局斥力**: 防止节点重叠，确保图谱在大规模数据下依然具有可读性。
- **关系拉力**: 有连接的节点会相互吸引，距离远近反映了关系的紧密程度。

---

## 5. 交互指南 (Interaction)

- **旋转**: 鼠标左键按住拖拽。
- **平移**: 鼠标右键按住拖拽。
- **缩放**: 鼠标滚轮。
- **聚焦**: **左键单击**任意节点，相机会平滑过渡并自动对焦该节点。
- **详情**: 悬停在节点上可查看完整 ID（含类型）。
