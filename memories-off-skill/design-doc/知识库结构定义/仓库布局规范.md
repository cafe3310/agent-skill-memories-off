# 知识库结构定义: Library Layout

本文档定义了 `memories-off` 所管理的知识库（Library）在本地文件系统中的标准目录结构和文件职责。

---

## 1. 顶层结构 (Top-level Structure)

一个知识库（Library）是一个独立的文件夹，其内部包含以下标准子目录和核心文件：

```text
[Library Root]/
├── .git/           # 版本控制与审计中心 (Git)
├── meta.md         # 知识库手册 (Manual)
└── entities/       # 实体目录 (Entities)
```

## 2. 核心组件说明 (Core Components)

### 2.1 知识库手册 (`meta.md`)
- **职能**: 存储知识库的全局元数据、规则、指导原则和类型定义。
- **操作原则**: 
  - 通过 `memocli explore` 一键获取手册内容及全库概览。
  - 通过 `memocli commit` 记录对此文件的手动编辑。

### 2.2 实体目录 (`entities/`)
- **职能**: 存放所有具体的知识实体 Markdown 文件。
- **命名规范 (Entity Name Standard)**: 
  - **允许字符**: 字母、数字、下划线及中文字符。
  - **标准化逻辑**:
    1. 非法字符（如空格、点、逗号、各种标点）统一替换为单个 `-`。
    2. 连续的 `-` 合并为单个 `-`。
    3. 移除首尾的 `-`。
  - **示例**: `Hello, World! .v2` -> `Hello-World-v2`。

### 2.3 版本控制与审计 (.git)
- **职能**: 通过 Git 记录每一笔变更。
- **操作**: 所有的修改工具（如 `append-update`, `create-entity`, `rename`）都内置了自动提交机制。Agent 手动修改文件后应运行 `memocli commit`。

## 3. 核心术语

- **Library Root**: 包含 `meta.md` 的根目录。
- **Entity Name (Slug)**: 实体文件的逻辑名称（不含 `.md` 后缀），经过标准化处理。
- **WikiLink**: 引用实体的格式 `[[Entity Name]]`。
