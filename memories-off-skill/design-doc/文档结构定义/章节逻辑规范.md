# 文档结构定义: Section Logic

本文档详细定义了 `memories-off` 中实体正文的章节组织逻辑及层级规则。

---

## 1. 标题层级契约 (Heading Hierarchy)

在 `memories-off` 中，为了确保知识图谱的扁平化、可解析性及未来“按需加载（Section Loading）”的稳定性，强制执行以下标题层级：

### 1.1 顶层实体名 (H1)
- **格式**: `# [Entity Name]`
- **约束**: 一个实体文件必须且只能包含一个 H1 标题，且必须位于正文的第一行。
- **职责**: 声明实体的本体名称。

### 1.2 业务章节 (H2)
- **格式**: `## [Section Title]`
- **约束**: 所有的业务分类、信息块、描述部分必须使用 H2 标题。
- **职责**: 作为知识块的原子容器，支持通过 `doctor` 自动标准化和未来 `load --section` 的精准获取。

### 1.3 废弃层级 (H3-H6)
- **约束**: 禁止在正式文档中使用 H3 及其更深层级的标题。
- **处理**: 系统在执行 `append-update` 或 `doctor` 治理时，会自动将匹配 `^#+\s` 的行（除 H1 外）统一转写为 H2。

---

## 2. 章节 (Section) 定义

一个 **Section** 被定义为一个逻辑单元，包含：
- **Heading**: H2 标题行。
- **Content**: 标题之下、直到下一个 H2 标题出现之前的所有文本行。

## 3. 唯一性与标准化

- **标题标准化**: 系统在处理标题时会去除多余空白，但建议人工书写时保持简洁清晰。
- **唯一性原则**: 建议在一个文档内保持 H2 标题的语义唯一性，以方便 Agent 进行精确的内容追加或查找。

## 4. 编辑行为准则 (Editing)

- **追加 (Append)**: 优先使用 `memocli append-update`。新内容会被包裹在带有双时间戳的 `UPDATE_BLOCK` 中。
- **拍平 (Flattening)**: 当向实体写入带有标题的内容时，系统会自动将所有标题层级映射为 H2。
- **治理 (Doctoring)**: 使用 `memocli doctor --rule-normalize-headers` 可以批量纠正历史遗留的凌乱层级。
