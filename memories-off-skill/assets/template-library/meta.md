# Knowledge Base Manual (meta.md)

## 1. 概述 (Overview)
这是一个遵循 `memories-off` 规范的模板知识库，用于演示标准结构、元数据 schema 和编辑准则。

## 2. 核心 Schema (Core Schema)
### 实体类型 (Entity Types):
- `人物`: 用于记录人物信息、偏好及与其相关的事件。
- `概念`: 用于记录抽象概念、定义及逻辑推演。
- `项目`: 用于追踪项目进度、目标和决策。
- `日志`: 记录按时间顺序排列的事件或思考。
- `宠物`: 记录家中宠物的健康与习性。
- `物品`: 记录家中资产、消耗品及购买记录。
- `对话记录`: 提炼的关键对话、观点或决策过程。
- `家务流程`: 沉淀的操作手册、清洁步骤或家庭规则。

### 元数据键 (Metadata Keys):
- `entity type`: **必须项且唯一 (Single Value)**。禁止使用逗号分隔多个分类。
- `date created`, `date modified`: 自动维护项。
- `status`: 生命周期（`草稿`, `稳定`, `归档`）。
- `aliases`: 别名，逗号分隔。

## 3. 关系定义 (Relations)
### 常用谓词 (Common Predicates):
- `relation as 成员`: 指向所属的组织或团队。
- `relation as 属于`: 指向其所属的父类或抽象概念。
- `relation as 负责人`: 指向该事物的核心管理者。
- `relation as 关联`: 通用的双向或单向关联。
- `relation as 包含`: 表示整体与部分的关系。

## 4. 命名与编辑准则
- 实体文件名：使用 `[类型]-[名称]` 或 `[名称]`，字符仅限 CJK、英文字母、数字和中横线。
- **关系值规范**：`relation as [谓词]` 后面的值必须使用**目标的完整实体名**（即文件名，如 `人物-张三`），不要包裹 `[[ ]]`。
- 章节引用：使用 `# 标题内容`，编辑时必须精确匹配。
