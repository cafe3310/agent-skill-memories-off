# 版本控制与审计中心 (Git)

## 1. 占位说明
在真实的 `memories-off` 知识库中，此位置应为真正的 `.git` 目录。在模板示例中，由于父目录本身已被 Git 托管，因此此处以 `[范例.git]` 目录进行形式展示。

## 2. 核心职能
- **审代**: 所有的变更历史通过 `git log` 查询。
- **备份**: 利用 Git 的快照机制实现版本安全。

## 3. 操作协议 (Protocol)
Agent 在对实体执行任何“写入”或“修改”操作后，必须调用 Git 工具执行提交：

1. **git add .**：暂存变更。
2. **git commit -m "[Action] [Target]: [Reason]"**：原子化提交并附带原因。

## 4. 提交信息参考
- `edit 人物-四盘: 更新了近期参与的项目列表`
- `create 宠物-咪咪: 新增猫咪基础档案`
- `rename 物品-渴望猫粮 -> 物品-Orijen: 统一使用英文原名`
